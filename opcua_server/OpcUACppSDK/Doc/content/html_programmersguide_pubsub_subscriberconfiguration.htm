Subscriber Configuration

Top  Previous  Next

A PubSub Subscriber consists of four main classes which need to be instantiated and configured:

•SubscribedDataSet
•DataSetReader
•ReaderGroup
•Connection

Message reception will proceed according to the parameters provided to these objects. Additional data type classes will be used in the configuration process.

SubscribedDataSet

The SubscribedDataSet defines the processing of the decoded DataSet in the Subscriber for a DataSetReader.

The relationship between a functional SubscribedDataSet and a functional DataSetReader is one to one.

Note: The SubscribedDataSet is a concrete base type. Application developers will typically use a specialized subtype instead to implement the dispatching of received DataSets.

See Configure SubscribedDataSet

 

DataSetReader

The DataSetReader is decoding the DataSetMessage content to a DataSet. For a successful decoding the DataSetReader has to be configured with the correct DataSetMetaData.

Note: Each DataSetReader is part of a ReaderGroup and it uses one SubscribedDataSet to dispatch the decoded DataSet.

See Configure DataSetReader

 

ReaderGroup

The ReaderGroup is used to group a list of DataSetReaders. It is not related to a particular NetworkMessage as the filter settings are configured on the DataSetReaders.

Note: A DataSetReader can be assigned to only one ReaderGroup.

See Configure ReaderGroup

 

Connection

The Connection class is used by both Publisher and Subscriber. For the Subscriber it is configured to receive NetworkMessages from the Message Oriented Middleware.

It configures parameters like the used protocol and the network address.

Note: A ReaderGroup can be assigned to only one Connection. The Connection has to be registered with the Application.

See Configure Connection
