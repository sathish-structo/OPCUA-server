Changed interface of Application::enableServerRegistration

Support of RegisterServer2 service

SDK now also supports IRegisterServer2 service for registration at LDS.
If LDS does not support this service, SDK uses RegisterServer as fall-back.

 

Support of ResendData method

ResendData is a well-known UA method at server object.
After invocation of this method a server shall re-send the newest values for all monitored items in a subscription in the next publish request(s).

SDK now implements this feature for server application; for client application a convenience method Client::Subscription::resendData is available.

 
Denial of repeated connect attempts

A server may monitor repeated attempts for denied user authentication. In case of repeated denied attempt it is likely that a client tries to guess user credentials.

To react on such attempts SDK provides the possibility to deny continued attempts from this host for a while.

See Application::configureRepeatedConnectAttempts and Server::Endpoint::configureRepeatedConnectAttempts for configuration of this feature.
