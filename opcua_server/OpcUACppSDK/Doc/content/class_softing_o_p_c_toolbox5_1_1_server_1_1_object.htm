The Object class is used to define an Object node in the address space of a server.

An Object is used to represent systems, system components, real-world objects and software objects in the address space of a server. An object consists of other objects, variables and/or methods.

Example: A device could be represented as an object whereas its process variables could be mapped as variables in the address space.

virtual void SoftingOPCToolbox5::Server::Object::handleCallRequest(Server::MethodCallTransaction * transaction, Server::MethodCallRequest * request) [virtual]

Callback method that is triggered when a UA method is invoked in context of this object by a client. This method should be overridden by server applications.

Parameters:

transaction

The transaction which contains the single call request

request

The call request which shall be handled.

Note:

The request is pre-processed by the SDK. In case of errors (e.g. access restrictions, input arguments, ...) the status code of the request is set. In case of invalid input arguments the input argument results is set by SDK.

When overriding this method you mustn't use blocking calls within this method since this blocks the UA communication.

This callback won't be invoked for methods where the callback Method::handleExecute() is overridden, unless the overriding implementation invokes Method::executeRequest().

The method does not have a return value - us request->setStatusCode to pass a result to the client.
