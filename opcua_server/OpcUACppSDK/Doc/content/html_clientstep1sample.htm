This topic belongs to Initializing and Terminating a Client Application and Connecting to a Server and Disconnecting. It refers to the sample application Client_1 and should be tested with the sample application Server_4. (See Programming Tutorial for where to find the tutorial samples)

Where to Find

Source code and project files belonging to the sample application are located in the directory <InstallDir>/Apps/Samples/Cpp/Tutorial/Client_1.

Running the Client

•The client uses an application instance certificate which must created and in the correct directory. See Setting up a Certificate Store for how to create the demo certificate store. The path to the certificate is specified at the top of Client_1.cpp, adapt it if it doesn't fit.
•Compile and start the sample applications Client_1 and Server_4. Run the client application in the debugger and step through the code.
•The client waits for keyboard input. Type "co" to connect to the running server and type "di" to disconnect. The procedure is similar in all tutorial steps -- new functionalities will be activated with new input commands.

Points of Interest

•The initialization and termination is done in _tmain(), runClient(), connect() and disconnect() in the file Client_1.cpp.
•The use of the function runClient() to ensure that all SDK objects that are created on the stack are released before terminating the application.
•The Application class. It is the "root class" for all client and server implementations. (See runClient() in the file Client_1.cpp)
•The use of the smart pointers and reference counted objects. (See runClient() in the file Client_1.cpp and Lifetime of SDK Objects)
•Use of namespaces: Client classes are defined in the SoftingOPCToolbox5::Client namespace.
•If you are not familiar with the standard template library (STL) yet, we recommended to have a closer look on vector and string classes.
•Setting the location of the client's application instance certificate.
•Establishing the connection is done in connect() and disconnecting in disconnect().
•Two ways to find and select matching endpoints are demonstrated using conditional compilation in connect(). To manually parse the endpoints and configure the session set USE_SESSION_CONFIGURE to 0 and to use the Session::configure service set it to 1.
