Setting up your Application for Remote Tracing

Top  Previous  Next

To enable TCP tracing in your application you have provide the SDK with the trace endpoint that you have configured previously.

// set up TCP trace to a listener on 127.0.0.1:1235, the message buffer is set to 100 msgs

Trace::instance()->enableTcpLogging(_T("127.0.0.1"), 1235, 100);

// allow traces for all trace groups and all levels

Trace::instance()->enableTraceLevel(EnumTraceGroup_All, EnumTraceLevel_All);

Writing you own trace messages

The following code illustrates how to use the SDK tracing mechanism to write trace messages for your own application:

// allow traces for all trace groups and all levels

Trace::instance()->enableTraceLevel(EnumTraceGroup_All, EnumTraceLevel_All);

Trace::instance()->writeLine(EnumTraceLevel_Error, EnumTraceGroup_User1, _T("Something is wrong!"), _T("ObjectID"));

Using the writeSourceLine() method, you can even include information about source files and lines:

Trace::instance()->writeSourceLine(EnumTraceLevel_Warning, EnumTraceGroup_User1, _T("Something is wrong but not that bad!"), _T(__FILE__), __LINE__);
