Setup Server Application

Top  Previous  Next

A server application does not require many additional configuration steps to support HTTPS, it only has to setup the PKI store configuration and to open an endpoint for the HTTPS protocol.

A basic security configuration is required, which means setting up a PKI store configuration.

PkiStoreConfiguration storeConfiguration;

storeConfiguration.setCertificateTrustListLocation(certificateTrustListLocation);

storeConfiguration.setIssuerCertificateListLocation(issuerCertificateListLocation);

storeConfiguration.setCertificateRevocationListLocation(certificateRevocationListLocation);

storeConfiguration.setRejectedCertificateLocation(rejectedCertificatesLocation);

storeConfiguration.setSuppressedCertificateChecks(certificateCheckOption);

storeConfiguration.setRevocationCheckOptions(crlCheckOption);

storeConfiguration.allowProxyCertificates(allowProxyCerts);

storeConfiguration.checkSelfSignedSignature(checkSelfSignedSignature);

pApp->setPkiStoreConfiguration(&storeConfiguration);

The endpoint can be opened similar to the TCP/IP protocol (see Providing Endpoints), the protocol in the beginning of the endpoint URL has to be different.

Instead of an endpoint URL like

"opc.tcp://localhost:4880/MyApplication/"

use e.g.

"https://localhost:443/MyApplication/"

 

 

Note: It is not required but recommended to use the default HTTPS port 443 for an endpoint, because this port is usually not blocked by networks and firewalls.

Note: The HTTPS certificate of the clients need to be in the trust list of the server PKI store to allow connections. Since most OPC UA applications reuse the Application Instance Certificate for HTTPS, there are usually no adoptions of the PKI store required.
