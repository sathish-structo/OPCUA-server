#include "ClientRedundantSubscription.h"

 

The RedundantSubscription class is used to publish data changes and/or events that occur in a server.

The redundant subscription class is similar to the subscription class but it is used within a redundant session and contains redundant monitored items.

EnumStatusCode SoftingOPCToolbox5::Client::RedundantSubscription::setLifeTimeCount(OTUInt32 lifeTimeCount)

Sets the threshold for the Subscription's life timer counter. Whenever the life time counter reaches the provided threshold, the server deletes the Subscription.

The server uses a life time counter to keep track of client activity for the Subscription. The life time counter of a Subscription is incremented by one whenever the publishing interval expires and it is reset to zero on every Subscription-related client activity (e.g. when the server receives a publish request). When the life time counter reaches the threshold provided by this method the server deletes the Subscription.

In order to ensure regular client activity in case the Subscription has no notifications to publish, the server publishes keep-alive messages when a given number of publishing intervals have elapsed without notifications being available at the server (see setMaxKeepAliveCount()). Therefore, the life time count depends on the maximum keep-alive count of the Subscription. The life time count should always be set to at least three times the maximum keep-alive count.

Parameters:

lifeTimeCount

When the publishing interval of the Subscription has elapsed this number of times, the Subscription will be deleted from the server. This value should be not less than three times the maximum keep-alive count. Please note that the server may revise the provided value.

See also:

setMaxKeepAliveCount() getRevisedLifeTimeCount()

Note:

The default value is the value 60.

Please use update() to apply changes to this parameter if the Subscription is already connected.

OTUInt32 SoftingOPCToolbox5::Client::RedundantSubscription::getMaxKeepAliveCount() const

The number of consecutive empty publish cycles after that the Server has to send a keep alive message.

EnumStatusCode SoftingOPCToolbox5::Client::RedundantSubscription::setMaxKeepAliveCount(OTUInt32 maxKeepAliveCount)

Sets the threshold for the Subscription's keep-alive counter. Whenever the keep-alive counter reaches the provided threshold, the server publishes a keep-alive notification message for this Subscription.

The server uses a keep-alive counter to inform the client that the Subscription is still alive and to provoke activity from the client in order to prevent the Subscription from being deleted (see setLifeTimeCount()). The keep-alive counter is incremented by one whenever the publishing interval expires and is reset to zero when the server publishes any notification message (including the keep-alive message) for the Subscription.

Parameters:

maxKeepAliveCount

The maximum value of the keep alive counter. When the counter reaches this threshold, the server publishes a keep-alive notification message for this Subscription and resets the counter. Please note that the server may revise the provided value.

See also:

setLifeTimeCount() getRevisedMaxKeepAliveCount()

Note:

The default value is the value 10.

Please use update() to apply changes to this parameter if the Subscription is already connected.

EnumStatusCode SoftingOPCToolbox5::Client::RedundantSubscription::setRePublishingEnabled(bool rePublishingEnabled)

Enables republishing in case of lost notifications.

RePublishing is a UA specific way to force the server to re-send lost notifications of a subscription. If republishing is enabled lost notifications are again requested from server (if available). By default republishing is disabled for a subscription.

It is recommended to enable republishing for subscriptions which need to receive every single notification (e.g. subscriptions receiving events, ...). For subscriptions which simply are interested in newest data values republishing is not necessary.

In case of lost notifications (either republishing disabled or server cannot provide lost notifications) the error EnumStatusCode_BadSequenceNumberInvalid is reported via the error callback.

EnumStatusCode SoftingOPCToolbox5::Client::RedundantSubscription::setPriority(unsigned char priority)

The relative priority of the Subscription. If several Subscriptions are ready to publish notification messages (Data changes, event notifications), the server uses the priority to decide which Subscription is allowed to publish first. In this case, the Subscription with higher priority will be allowed to publish its notification messages first.

Use priority value "0" if the Subscription does not require special priority settings (recommended!).

Note:

The default value is 0.

Please use update() to apply changes to this parameter if the Subscription is already connected.

EnumStatusCode SoftingOPCToolbox5::Client::RedundantSubscription::conditionRefresh(OTUInt32 redundantServerIndex)

Refreshes the current state of all monitored conditions

The Subscription must be connected when this method is called, otherwise an error code will be returned. If successful, the subscription will receive notifications for all event monitored items. For each event monitored item in the subscription the following sequence of notifications shall be received:

•An event of type RefreshStartEventType
•The current status of all retained conditions notifying the monitored item
•An event of type RefreshEndEventType

The notification of events can start before the method returns.

Returns:

EnumStatusCode_Good if refresh was successful. EnumStatusCode_BadInvalidState if session is not connected EnumStatusCode_BadSubscriptionIdInvalid if subscription is unknown in server EnumStatusCode_BadRefreshInProgress if another refresh of this subscription is already in progress

See also:

onEventsReceived(), MonitoredItem()

virtual void SoftingOPCToolbox5::Client::RedundantSubscription::onEventsReceived(const std::vector< Client::RedundantMonitoredItem * > & items, const std::vector< EventFieldList > & values, OTUInt32 redundantServerIndex) [virtual]

Callback for reported events. This function must be overridden in a derived class to obtain event reports

The reported events are filtered by the monitored items where clause of the filter. The received event fields match the specified select clause of the filter.

In case of a queue overflow of a monitored item in the server, an event of type RefreshRequiredEventType is reported. It indicates that due to queue overflow some events are missing. An application can perform a conditionRefresh to update the current states of the monitored conditions. The reporting of this event type ignores any filtering.

Parameters:

items

The MonitoredItems that have received events. This vector may contain a NULL pointer if the client receives a notification for a MonitoredItem has has been already removed.

values

The received events. The number of elements in this vector matches the number of notified items.

redundantServerIndex

The index of the server, specified by setUrls() of the session.

Note:

The EventFields in values can be null if the selected value is not accessible, e.g. if not all events have a selected property

See also:

conditionRefresh()

virtual void SoftingOPCToolbox5::Client::RedundantSubscription::onError(EnumStatusCode status, OTUInt32 redundantServerIndex) [virtual]

Error callback. This must be overridden in a derived class in order to receive error notifications.

Typical contexts in which this callback can be called are errors in following situations:

•creating the Subscription on the server (connect)
•deleting the Subscription
•setting the publishing mode
•modifying the Subscription
•processing notification message or publish response from server
•notification messages from server were missing

Parameters:

status

The error condition

redundantServerIndex

The index of the server, specified by setUrls() of the session.

virtual void SoftingOPCToolbox5::Client::RedundantSubscription::onStatusChange(bool status, OTUInt32 redundantServerIndex) [virtual]

Connection status change callback. This callback will be called whenever the connection status of the Subscription changes.

Parameters:

redundantServerIndex

The index of the server, specified by setUrls() of the session.
