The implementation of an OPC UA client application most likely involves four classes which are depicted in the class diagram above: Application, Session, Subscription and MonitoredItem.

The Application class is a singleton that contains application-wide configuration information and manages Session instances. A Session represents a connection with an OPC UA server and exposes methods to request services such as reading or writing data from/to a server. When a client wants to monitor data changes or receive events from a server, it needs to create Subscriptions and MonitoredItems at the server. While a MonitoredItem contains the identifier of a node that is being monitored and defines the interval at which a node's value is being sampled, a Subscription bundles several MonitoredItems and defines the interval at which changed data is being reported to the client.
