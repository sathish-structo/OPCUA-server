Server Containment Hierarchy

On server side many of the items defined by the OPC UA specifications can be found as class implementations.

So all object types for Nodes are represented by a class as well as e.g. a Session class / Endpoints etc..

The client side is more data oriented and has only a small set of classes required for Session and Subscription handling.

The following diagram shows the parent relations hierarchy for the SDK server classes:

A server application can have n Endpoints with again n Sessions (a Session is created when a client connects to an Endpoint).

A Session can have n Subscriptions with again n MonitoredItems where each of these items is related to one BaseNode.
