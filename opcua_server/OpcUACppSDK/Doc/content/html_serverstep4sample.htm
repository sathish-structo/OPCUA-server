This topic belongs to Creating a Custom UA Method. It refers to the sample application Server_4 and should be tested with the sample application Client_6. (See Programming Tutorial for where to find the tutorial samples)

Where to Find

Source code and project files belonging to the sample application are located in the directory <InstallDir>/Apps/Samples/Cpp/Tutorial/Server_4.

Running the Server

•Compile and start the sample applications Server_4 and Client_6. Run the server application in the debugger and step through the code.
•After connecting the client with the command "co" and browsing the address space to get the necessary NodeIds with "br", use the command "op" (open) and "cl" (close) to call the Method with a different input argument to perform an action. These commands also have optional parameters to specify an instance of a simulated device.

Points of Interest

•The related steps are implemented in Refrigerator::initialize(), Refrigerator::handleCallRequest(), Refrigerator::openDoor() and Refrigerator::closeDoor().
•How the Method is configured and added to the RefrigeratorType. (See Refrigerator::initialiaze())
•How to handle the MethodCallRequest with an overridden handleCallRequest of the class Object. (See Refrigerator::handleCallRequest() for handling the call and Refrigerator::openDoor and Refrigerator::closeDoor() for method handling)
