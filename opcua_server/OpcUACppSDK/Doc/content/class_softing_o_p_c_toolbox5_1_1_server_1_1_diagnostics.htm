Diagnostics Class Reference

Top  Previous  Next

#include "ServerDiagnostics.h"

 

This class controls the collection of diagnostic information in the server. Diagnostic information is exposed to clients through the Server object in the server's address space.

By default, collecting of diagnostic information is disabled and can only be enabled by a server application.

Note:

Enabling diagnostics will increase the server load.

EnumStatusCode SoftingOPCToolbox5::Server::Diagnostics::setEnabled(bool enable)

Enables or disables collecting of diagnostic information in the server. Diagnostics are disabled by default.

Parameters:

enable

If true, the server collects diagnostic information.

EnumStatusCode SoftingOPCToolbox5::Server::Diagnostics::setEnabledFlagWritable(bool writable)

Allows clients to enable or disable collecting of diagnostic information.

Parameters:

writable

If true, clients may enable or disable diagnostics by writing to the "EnabledFlag" property of the diagnostics object in the address space.

OTUInt32 SoftingOPCToolbox5::Server::Diagnostics::getUpdateInterval() const

Returns the interval (in milliseconds) that the server uses to update diagnostic information in the address space.

EnumStatusCode SoftingOPCToolbox5::Server::Diagnostics::setUpdateInterval(OTUInt32 updateInterval)

Sets the interval (in milliseconds) that the server uses to update diagnostic information in the address space. The default update interval is 1000ms. Please refer to Application for reading or setting minimum and maximum values for this parameter.

Parameters:

updateInterval

The update interval

See also:

Application::setMinDiagnosticsUpdateInterval() Application::setMaxDiagnosticsUpdateInterval()
