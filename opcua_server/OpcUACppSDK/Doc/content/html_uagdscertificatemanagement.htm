Certificate Management

Top  Previous  Next

The certificate management of the GDS is responsible to create, update and distribute certificates and trust lists to the registered applications.
This ensures that the involved certificates don't expire and allows to configure the trust relation between applications from one central location.
The created certificates are typically signed by a Certificate Authority (CA) which is also maintained by the Certificate Manager.

This can simplify certificate management even in medium to small OPC UA environments. Different implementations are expected.

Note: The Certificate Manager is usually a GDS, but it is also possible that another OPC UA application acts as Certificate Manager without supporting the other features of a GDS.

 

The managed certificates and trust lists can be for

•application instance certificates
•user certificates for x509 authentication
•HTTPS certificates
•any custom certificates and trust lists

 

The Certificate Manager organizes the trust lists of all UA applications as certificate groups. A single UA application can have zero to many certificate groups and multiple UA applications can also share the same certificate groups (e.g. DefaultApplicationGroup).

Each trust list / certificate group consists of the following contents:

•trusted certificates
•certificate revocation lists (CRLs) of trusted certificates
•issuer certificates (certificates which have signed other certificates and which are required to resolve a certificate chain, but not all these signed certificates shall be trusted)
•certificate revocation lists (CRLs) of issuer certificates

 

The OPC Foundation provides two information models to manage certificates and trust lists of clients and servers.

•Pull Management: This model is mainly targeted for UA client applications. The UA application takes the active role to retrieve the certificates and trust lists from the Certificate Manager.
•Push Management: This model is targeted for UA server applications where a Certificate Manager takes the active role to distribute certificates and trust lists.
