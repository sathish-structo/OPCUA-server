This topic belongs to Creating Custom Types. It refers to the sample application Server_3 and should be tested with the sample application Client_6. (See Programming Tutorial for where to find the tutorial samples)

Where to Find

Source code and project files belonging to the sample application are located in the directory <InstallDir>/Apps/Samples/Cpp/Tutorial/Server_3.

Running the Server

•Compile and start the sample applications Server_3 and Client_6. Run the server application in the debugger and step through the code.
•After connecting the client with the command "co" use the command "br" to browse the address space for available refrigerator instances. To read and display the values of the user defined nodes use the command "rd" and use "wr" to write the setpoint temperature. These commands also have optional parameters to specify an instance of a simulated device or a set value. To simulate the actual temperature use the command "sim" in the server console.

Points of Interest

•The related steps are implemented in Refrigerator::initialize(), Refrigerator::create(), Refrigerator::createFolder(), RefrigeratorVar::handleReadRequest(), RefrigeratorVar::handleWriteRequest(), Refrigerator::updateSdkCache() and initAddressSpace().
•How to create a custom ObjectType (by inheriting from Object), set it on the type tree and create instances of this type using init(). (See Refrigerator::initialize() and Refrigerator::create())
•How to create an object of "FolderType" to organize the created instances. (See Refrigerator::createFolder() and Refrigerator::create())
•How to fill the AddressSpace with different objects and object types. (See initAddressSpace())
•How to handle ReadRequests and WriteRequests of components, i.e. how to read values from a device or write values on a device. (See RefrigeratorVar::handleReadRequest() and RefrigeratorVar::handleWriteRequest())
•How to check if the value to write is within a desired InstrumentRange of an AnalogItem. (See RefrigeratorVar::handleWriteRequest())
•How to simply update the component values in the SDK cache. (See Refrigerator::updateSdkCache())

Note:

The Refrigerator::simulate() method is just a very simple simulation to demonstrate data change and it does not have to be analysed to understand the functionalities of the SDK.
