Overview of the SDK API

The OPC UA C++ Server & Client SDK provides an API by a set of classes that can be used to develop both OPC UA-enabled client and server applications. Depending on whether you want to develop a server, a client or both, you may only be interested in a subset of the SDK API.

The SDK API is organized in namespaces which are depicted in the following figure:

The SDK defines the three namespaces SoftingOPCToolbox5, SoftingOPCToolbox5::Server and SoftingOPCToolbox5::Client and adds types to the global namespace.

•The SoftingOPCToolbox5 namespace comprises classes that can be used by both client and server applications. This namespace, for example, contains the Application class which represents an OPC UA application and is the root class of the SDK. Structured OPC UA data types are also defined within this namespace. The DataValue class, for example, stores a value, a quality and a timestamp.
•The SoftingOPCToolbox5::Client namespace consists of classes that are only interesting to client applications. Client applications may use the Client::Session class, for example, to connect to OPC UA servers.
•The SoftingOPCToolbox5::Server namespace encompasses classes that are only of interest to server applications. For example, this namespace contains classes that represent nodes in the address space such as Server::Object or classes that represent connections from clients (Server::Session).
•The global namespace comprises operating system independent definitions for fundamental types (such as OTInt32 for an signed 32-bit integer), string constants and enumerated types.

Note:

The namespaces SoftingOPCToolbox5::Client and SoftingOPCToolbox5::Server contain classes with the same name such as Client::Session and Server::Session. Please ensure to use fully qualified names for server and client classes if you develop an application that uses both the server and the client API.
