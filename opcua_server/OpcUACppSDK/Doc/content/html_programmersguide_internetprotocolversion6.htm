Internet Protocol version 6

Top  Previous  Next

The Internet Protocol version 6 (IPv6) is designed to displace the IPv4. With the success of the internet the number of IPv4 (32 bit) addresses became rare. The IPv6 enlarges the theoretical address space from 232 (≈ 4,3 billion 4,3 109) addresses to 2128 (≈ 340 Undecillion 3,4 * 1036). The Network Address Translation NAT proceeding (because of shortage of addresses in IPv4) conflicts with the PointToPoint-Connection as a central design principle of the internet. With IPv6 this enough addresses are available that NAT is not necessary anymore.

 

Notation

IPv4

The notation of the IPv4 Address is a combination of 4 octets (bytes) e.g. C0:A8:13:7B. This is written in the dotted decimal notation like 192.168.19.123. A endpoint URL looks like:

opc.tcp://192.168.19.123:4880/softing

IPv6

The decimal format for an IPv6 address (ddd.ddd.ddd.ddd.ddd.ddd.ddd.ddd.ddd.ddd.ddd.ddd.ddd.ddd.ddd.ddd) is hard to read and unmanageable, so the the representation is using a hexadecimal notation, which is additionally simplified by combining any two octets to one block like 2001:012f:0000:0000:0000:0000:ab12:1234. Leading 0 can be omitted in to shorten the address (2001:12f:0:0:0:0:ab12:1234). In addition to that it is possible to replace contiguous 0-blocks with a "::", but only once in an address to be unambiguously, so the final simplified address looks like this: 2001:12f::ab12:1234. For using the IPv6 address in an URL it is surrounded by square brackets to distinguish between the separator of the octet-blocks and the following port number. So the loop back address 0000:0000:0000:0000:0000:0000:0000:0001 becomes ::1

So a endpoint URL is written like:

opc.tcp://[2001:12f::ab12:1234]:4880/softing

 

Certificates

How to create a certificate store is described here: Setting up a Certificate Store. On Microsoft Windows, using the create_demo_store batch, it is recommended that the manual input is used.

> cd <InstallDir>/Source/PKI

> create_demo_store.bat -c <hostname> <domainname> <ip-address>

For an IPv6 address use the above described notation without the square brackets and without any suffixes for the net notation like "/64".

 

Limitations

On Microsoft Windows machines it is not possible to restrict an endpoint using an IPv6 address to a single Interface for now.

endpoint->setRestrictToSingleInterface(true);
