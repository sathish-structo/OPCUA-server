The RedundantMonitoredItem class is used to monitor data changes or events in a server.

This class is similar to the MonitoredItem class but is used within a redundant session and a redundant subscription.

EnumStatusCode SoftingOPCToolbox5::Client::RedundantMonitoredItem::addItemsToReport(const std::vector< Client::RedundantMonitoredItemPtr > & itemsToReport)

Adds a list of item that shall be reported when this monitored item triggers.

OPC UA allows setting up monitored items that report their data changes or events only if they are triggered by some other monitored item. The triggering mechanism is a useful feature that allows clients to reduce the data volume on the wire by configuring some monitored items to sample frequently but only report when some other event happens.

The monitoring mode of the items to report (i.e. triggered items) is set to sampling-only so that it will sample and queue notifications without reporting them.

The following triggering modes are defined:

•If the monitoring mode of the triggering item is SAMPLING, then it is not reported when the triggering item triggers the items to report.
•If the monitoring mode of the triggering item is REPORTING, then it is reported when the triggering item triggers the items to report.
•If the monitoring mode of the triggering item is DISABLED, then the triggering item does not trigger the items to report.
•If the monitoring mode of the item to report is SAMPLING, then it is reported when the triggering item triggers the items to report.
•If the monitoring mode of the item to report is REPORTING, this effectively causes the triggering item to be ignored. All notifications of the items to report are sent after the publishing interval expires.
•If the monitoring mode of the item to report is DISABLED, then there will be no sampling of the item to report and therefore no notifications to report.

Parameters:

itemsToReport

A list of monitored items that shall report their notifications when this monitored item triggers.

Note:

Please note that both the reporting monitored items and the triggering monitored item need to be connected to the server (i.e. isConnected() must return true).

See also:

EnumStatusCode SoftingOPCToolbox5::Client::RedundantMonitoredItem::setIndexRange(const INumericRange * pIndexRange)

Sets the index range that the server applies to DataValues that contain variants of type 'array'. The index range allows clients to request only a certain range of an array.

Examples:

•"1" - would select the second array element only
•"1:2" - would select the second and third elements in an array
•"1:2,0:1" - would select an range of a matrix.
•"1,2" - would select the element at position [1,2] in the matrix

Parameters:

pIndexRange

The index range to apply to arrays

Note:

The index range cannot be modified after the RedundantMonitoredItem has been connected.

The second argument of the range specification for a dimension must be greater than the first one. The following values are, for example, invalid: "1:1", "2:1"

EnumStatusCode SoftingOPCToolbox5::Client::RedundantMonitoredItem::setMonitoringMode(EnumMonitoringMode monitoringMode)

Sets the monitoring mode of the monitored item at the primary server. The monitoring mode determines whether the monitored item collects notification messages (i.e. data change notifications or event notifications) and, if so, whether the Subscription publishes the collected notifications or not.

The monitoring mode is usually set before the monitored item is being connected. Changing the monitoring mode of a connected monitored item will immediately change its state in the server.

Parameters:

monitoringMode

The monitoring mode of the monitored item.

Note:

Default value is EnumMonitoringMode_Disabled, i.e. notification messages are neither collected nor reported to the client. Use EnumMonitoringMode_Reporting to enable reporting of notification messages.

To modify the monitoring mode of multiple items, please use the methods Client::RedundantSubscription::setMonitoredItemsMonitoringMode() or Client::RedundantSubscription::setAllMonitoredItemsMonitoringMode().

double SoftingOPCToolbox5::Client::RedundantMonitoredItem::getSamplingInterval() const

See also:

EnumStatusCode SoftingOPCToolbox5::Client::RedundantMonitoredItem::setSamplingInterval(double samplingInterval)

Sets the sampling interval of the monitored item. The server may revise this value to a more appropriate value which can be retrieved by getRevisedSamplingInterval() after the monitored item has been connected.

The value 0 indicates that the server should use the fastest practical rate. The value -1 indicates that the publishing interval of the subscription shall be used for the sampling interval.

In case of an event monitored item this value must be 0.

Parameters:

samplingInterval

The sampling interval in milliseconds

Note:

The default value is 0.

Please use Client::RedundantSubscription::updateChangedMonitoredItems() to apply changes to the sampling interval when the monitored item is already connected (isConnected() == true).

OTUInt32 SoftingOPCToolbox5::Client::RedundantMonitoredItem::getQueueSize() const

Returns the maximum number of data change or event notifications that a monitored item may hold in its queue. Notifications are dequeued by the Subscription every publishing interval.

See also:

setQueueSize()

EnumStatusCode SoftingOPCToolbox5::Client::RedundantMonitoredItem::setQueueSize(OTUInt32 queueSize)

Sets the maximum number of data change or event notifications that a monitored item may hold in its queue. Notifications are dequeued by the Subscription every publishing interval.

The queue of notification messages allows clients to collect more than one data value change or event for the same node with one publish call.

If the server wants to enqueue a notification message and the queue has reached the maximum number of elements, then depending on the DiscardOldest parameter (see setDiscardOldest()) either the oldest or the latest notification is discarded.

Parameters:

queueSize

The maximum number of data change or event notification messages that the monitored item shall keep in its queue before the queue is emptied by the Subscription during publishing. If the value 0 is provided, the server will use the default queue size.

Note:

The default value is the value 0 which means that the server will use the default queue size.

Please use Client::RedundantSubscription::updateChangedMonitoredItems() to apply changes to the queue size when the monitored item is already connected (isConnected() == true).

bool SoftingOPCToolbox5::Client::RedundantMonitoredItem::getDiscardOldest() const

Returns whether the oldest or the latest notification message is discarded when the queue of notification messages is full and a new notification is to be enqueued.

Returns true, if the oldest notification message is discarded. Returns false, if the latest notification message (i.e. the new notification to enqueue) is discarded.

See also:

•which DataValue attributes (value, quality/status, timestamp) must have changed for the data change to be enqueued.

virtual void SoftingOPCToolbox5::Client::RedundantMonitoredItem::onError(EnumStatusCode errorCode, OTUInt32 redundantServerIndex) [virtual]

This method is called whenever an error concerning this monitored item occurs. Applications should override this method to catch errors that occur due to this monitored item.

Typical contexts in which this callback is called are errors in following situations:

•connecting the monitored item on the server node
•deleting a monitored item from the server
•updating the server with the monitored items modifications
•setting the monitoring mode for monitored items
•setting triggered monitored items (for the items to be added or removed)

Parameters:

errorCode

The status code of the error that occurred

redundantServerIndex

The index of the affected server (see Client::RedundantSession::setServerInformation()).
