Programming Samples

The programming samples cover specific topics of OPC UA functionality and how to realize this functionality in OPC UA C++ Server & Client SDK V6.00.

The samples usually contain both server and client applications, either to show the topics on both sides or to have the opposite application for testing purposes.

 

Overview on the Programming Samples

•Secure Communication: Offering and establishing a secure communication
•Discovery Service: Register a server at the local discovery server. Use the discovery server from a client to find UA servers
•Limited Resources: Limit the usage of resources at server
•User Access Control: Identification of different users and providing different access rights
•Asynchronous Access: Usage of asynchronous requests on client
•Audit Events: Enabling audit events on server and receiving the events on client
•NodeSetModelImport: Importing a Nodeset from a XML file
•ComplexDataType: Shows how to define and use a custom complex data type on a server application.
•File Transfer: Access individual files on the server using FileType ObjectType
•PubSub: Publish and subscribe data with connectionless PubSud via UDP (uni-cast, multi-cast or broad-cast).
•Virtual Nodes Server: Handle service requests to nodes which are not stored in the SDK.

 

Where to Find the Sample Application Projects and How to Start them

Windows:

•See Windows Installation Directory Structure for where to find the sample projects (Programming Samples). In the folders of the samples there is each a solution file for Visual Studio 2013 to 2019 (e.g. Tutorial_20[13|15|17|19|22].sln) containing a pair of a client and server application.
oOpen the appropriate file with Visual Studio.
oCompile and start/debug the projects with Visual Studio.

Linux:

•Please follow the following instructions to compile and run the sample applications on Linux:
oSee Linux Installation Directory Structure for where to find the sample projects (Programming Samples) and executable files (the same path as for the Test Applications).
oOpen a command shell.
oChange the working directory to the path of the tutorial.
oCompile the desired sample by calling "make -f linux_gcc.mak".

VxWorks:

•When you have integrated the SDK into the Wind River Workbench, please follow the instructions below. Otherwise, please follow the instructions on how to create a new project for the sample applications.
oSelect the menu "File" -> "New" -> "Example".
oSelect "VxWorks Real Time Process Sample" and select "Next".
oSelect the desired Softing OPC UA SDK sample and select "Finish".
oCompile and start/debug the desired projects with the Wind River Workbench.
