This topic belongs to Initializing and Terminating a Server Application and Providing Endpoints. It refers to the sample application Server_1 and should be tested with the sample application Client_6. (See Programming Tutorial for where to find the tutorial samples)

Where to Find

Source code and project files belonging to the sample application are located in the directory <InstallDir>/Apps/Samples/Cpp/Tutorial/Server_1.

Running the Server

•The server needs an existing application instance certificate and it must be in the correct directory. See Setting up a Certificate Store for how to create the demo certificate store. The path to the certificate is specified at the top of Server_1.cpp, adapt it if it doesn't fit.
•Compile and start the sample applications Server_1 and Client_6. Run the server application in the debugger and step through the code.
•The client waits for a keyboard input. Type "co" to connect the client to the running server and type "di" to disconnect it. The other client commands are not supported for this server. The procedure is similar in all tutorial steps -- further server implementations can be tested with additional client commands.

Points of Interest

•The initialization and termination is done in tmain() and runServer() in the file Server_1.cpp, configuring and adding the endpoint in addEndpoint().
•How to prepare a project to use the SDK. (See Setting up a Project)
•The Application class. You get the instance of this singleton by the call Application::instance(). The Application class is the root class for all client and server implementations. (See runServer() in the file Server_1.cpp)
•The usage of the smart pointers and reference counted Objects. (See runServer() and addEndpoint() in the file Server_1.cpp)
•How to load the certificates. If you use a different location for storing certificates you have to adapt the directories. (See runServer() in the file Server_1.cpp)
•The Endpoint configuration.
•The shutdown code.
•Use of namespaces: Server classes are defined in the Server namespace, to distinguish it from the client classes, which are in the Client namespace. You have to use fully qualified names if your application needs to use classes from both namespaces in the same source file.
•Get familiar with using STL container classes. The C++ API makes extensive use of STL-container and string classes.
