EnumStatusCode SoftingOPCToolbox5::Server::HistoryReadEventContinuationPoint::setContextFromTimestamp(const IDateTime * timestamp)

A proposed interface to store context information to be able to continue an incomplete request in a follow up request.

It is proposed to store the timestamp of the next HistoricalEvent where the request shall be continued.

Note:

This method isn't used by the SDK and is just a proposal and provides a usable default implementation. To store additional or different context information for this ContinuationPoint, create and use a derived HistoryReadEventContinuationPoint which holds the additional data as additional members.

EnumStatusCode SoftingOPCToolbox5::Server::HistoryReadEventContinuationPoint::setContextFromTimestamp(const DateTime & timestamp)

const IDateTime * SoftingOPCToolbox5::Server::HistoryReadEventContinuationPoint::getStartTime() const

Returns the beginning of the period to read. If the start time equals the minimum value of DateTime, no specific start time is specified.

Note:

The client must have specified at least two of the three parameters StartTime, EndTime and MaxNumberOfValuesPerNode. This precondition is already verified by the SDK.

Calling this getter is only valid after this ContinuationPoint was assigned to a HistoryReadEventRequest

OTUInt32 SoftingOPCToolbox5::Server::HistoryReadEventContinuationPoint::getMaxNumberOfValuesPerNode() const

Returns the maximum number of historical values to return for every node in a request.

If the available number of values exceeds the provided maximum, the server application may create a continuation point to allow the client to fetch more values.

Note:

The client must have specified at least two of the three parameters StartTime, EndTime and MaxNumberOfValuesPerNode. This precondition is already verified by the SDK.

Calling this getter is only valid after this ContinuationPoint was assigned to a HistoryReadEventRequest

EnumStatusCode SoftingOPCToolbox5::Server::HistoryReadEventContinuationPoint::getFilter(EventFilter & filter) const

Returns the filter that shall be used by the server application to determine which historical events to include in the response and which event fields to return. This parameter must be specified and at least one EventField must be selected.

EnumTimestampsToReturn SoftingOPCToolbox5::Server::HistoryReadEventContinuationPoint::getTimestampsToReturn() const

Returns which timestamps to return for DataValues in event fields. This parameter can be ignored for event fields that are not of type DataValue.

Note:

This parameter applies only to event fields that are of type DataValue.
