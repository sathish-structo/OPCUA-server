SupressedOrShelved

The SupressedOrShelved property of the AlarmCondition determines if a alarm condition is either suppressed or shelved. If this property is set to true, only actions initiated by client (e.g. acknowledge) or changes in the SupressedOrShelved shall be notified to the client.

The value can .

The value of that property be retrieved by the getSuppressedOrShelved method and is determined by the:

•SupressedState - allows a server to suppress the retorting of an alarm in certain conditions
•ShelvingState - allows the alarm to be shelved by a client

Note:

It is not advisable to change this value directly since it is computed out of two different sources - the suppressed flag and the shelving state.

SuppressedState Property

The SupressedState property can be used to suppress alarm reporting to clients.

Changing the SupressedState property is done via the AlarmCondition::suppress method, which is initiated by the server application due to specific reasons. Such reasons might be for example to avoid alarm notification related to equipment that is shutdown, or machineries that are currently not in use.

Note:

Do not change the value of the corresponding variable directly. Direct changes shall bypass maintaining other properties and might avoid necessary reporting.

ShelvingState

A client can suppress annoying alarm notification by shelving specific alarms - if the server supports this since shelving of alarms is optional.

If the server supports shelving a ShelvedStateMachine is available as a component of the alarm. This state machine exposes 3 UA methods to the client:

•shelve: Sets the alarm into a "one-shot-shelved" state. No notifications are reported until the alarm switches between active and inactive.
By shelving an alarm using this method e.g. notification of changes between High and HighHigh states can be suppressed.
•timeShelve: Shelves the alarm for a specific duration. The maximum duration can be specified by the server
•unshelve: Removes all shelving states. Every change in the alarm state is reported to the client

Independent from shelving changes initiated by client (method calls) are always reported.

For more details see the documentation of ShelvedStateMachine. The shelved state will automatically clear when the alarm gets inactive.
