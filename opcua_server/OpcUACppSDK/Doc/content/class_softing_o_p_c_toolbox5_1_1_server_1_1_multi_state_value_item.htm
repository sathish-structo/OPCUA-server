MultiStateValueItem Class Reference

#include "ServerMultiStateValueItem.h"

 

This class represents a DiscreteItem that can have more than two states and where the state values does not consist of consecutive numeric values (may have gaps).

The MultiStateValueItem can contain values of any numeric data type.

Properties of MultiStateValueItem:

•EnumValues (mandatory): A list of pairs which map specific discrete values to LocalizedText. Example: {"1/OPEN", "2/CLOSE", "4/IN TRANSIT"}.
•ValueAsText (mandatory): A property showing the LocalizedText matching the corresponding value.

Sets the actual state of the MultiStateValueItem.

The property "ValueAsText" is updated automatically.

Parameters:

The new state set to this item. The value is provided as Value since the instance can have any numeric data type. The value is validated that a defined state for this value exists.
