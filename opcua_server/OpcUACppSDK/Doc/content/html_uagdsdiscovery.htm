Discovery

Top  Previous  Next

An OPC UA application can query the GDS to find other OPC UA applications of interest from the administrative domain.

OPC UA applications can invoke the discovery on the GDS via UA Method calls.
The GDS exposes the UA Method QueryApplications where the calling application can provide some filters, this allows to either search for a certain application, e.g. via the Application URI, or to search in a more generic way, e.g. to search all servers which support a certain server capability or for all servers with the same Product URI.

Typically only client applications use the GDS Discovery to search for matching server applications, but it is also possible that server applications search for other UA applications, e.g. to find client applications for ReverseConnect or to find other redundancy partners which expose the same data as the own server.

Note: Besides the QueryApplications method there is also a deprecated QueryServers method. The QueryServers method is only usable to search for servers, the method may still be used by older UA clients but new OPC UA applications should prefer to use the QueryApplications method.

 

A searching client application can discover and connect to a yet unknown server in the following approach:

•Establish a session to the known endpoint URL of the GDS of the current domain
•Call the OPC UA Method QueryApplications to search for matching applications
•Check the reported ApplicationDescriptions and retrieve the contained Discovery URL(s) of the server
•Invoke the GetEndpoints service with the selected Discovery URL of the server to retrieve the server's EndpointDescriptions
•Select a desired endpoint configuration, e.g. with message security and certain user token policies
•Establish a session to the selected endpoint URL of the selected server.
