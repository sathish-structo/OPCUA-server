Running the Trace Viewer

Top  Previous  Next

Every time your application connects to one of the trace endpoints that you have configured previously, a new node is added below this endpoint. A green icon next to the node indicates the latest trace run while a red icon denotes a previous trace run.

Output of the Trace Viewer

Each line in the Trace View contains the following information:

•Time: The time stamp when the line was created.
•Thread: The thread ID of the thread where the line was generated.
•Source: The trace line's "functional source" (see below).
•Text: The trace line's text.
•Source File / Location: The trace line's source information (if available) -- usually the source file and line where the trace entry was created.

Available Trace Sources

The source describes the origin of the trace line. The following sources produce trace messages:

•API: A trace line documenting an API call.
•Core: A trace line coming from the SDK Core.
•Stack: A trace line coming from the OPC UA Stack.

Severity Information

Each trace line has a severity (displayed by different colors) which can be one of the following:

•Fatal Error: An severe error that is usually not recoverable. A restart of the application is recommended.
•Error: An error that usually does not require a restart of the application.
•Warning: A warning indicates potential issues in the code that should be investigated.
•Info: An informational line about an action or status change.
•Debug: A debug information line.
