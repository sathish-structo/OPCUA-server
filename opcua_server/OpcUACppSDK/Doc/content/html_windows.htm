Installing the SDK for use with Microsoft Windows

Please run the provided setup InstallOpcUaCppSDKWindows6.00.0.exe and follow the instructions.

 

Installation Directory

The installation directory of the SDK, hereafter referred to as <InstallDir>, should not contain special characters like blanks, "#", ...
The SDK itself works well in such installation paths; in case of source code license you will run into issues since OpenSSL does not support these special characters.

The <InstallDir> contains the following directories:

Directory location

Description

<InstallDir>/Doc

Location of the SDK's documentation

<InstallDir>/Source

Location of SDK libraries, test applications and programming samples.

<InstallDir>/Tools

Location of additional utilities (trace viewer)

SDK Components (Binary)

For the Windows platform, the SDK's components are shipped as shared libraries (DLLs). The libraries are available for both 32 bit and 64 bit CPU architectures.

32 bit libraries:

•TB5CPP.dll: The TB5CPP component (C++ API)
•TB5OT.dll: The TB5OT component (SDK core)
•TB5STACK.dll: The TB5Stack component (OPC UA communication stack)
•TB5UTIL.dll: The TB5Util component (Utilities library)
•libcrypto-1_1.dll: The OpenSSL component for cryptography
•libssl-1_1.dll: The OpenSSL component for SSL/TLS
•libxml2.dll: The libxml component

64 bit libraries:

•TB5CPPx64.dll: The TB5CPP component (C++ API)
•TB5OTx64.dll: The TB5OT component (SDK core)
•TB5STACKx64.dll: The TB5Stack component (OPC UA communication stack)
•TB5UTILx64.dll: The TB5Util component (Utilities library)
•libcrypto-1_1.dll: The OpenSSL component for cryptography
•libssl-1_1.dll: The OpenSSL component for SSL/TLS
•libxml2x64.dll: The libxml component

The libraries can be found at the following locations. The placeholder <PlatformDir> resolves to Win32 for 32 bit binaries and x64 for 64 bit binaries.

Directory location

Description

<InstallDir>/Source/bin/<PlatformDir>/VS20[13|15|17|19|22]/Debug

Location of DLLs that are built with Visual Studio 20[13|15|17|19|22]. The DLLs contain debug information and use the multi-byte character set.

<InstallDir>/Source/lib/<PlatformDir>/VS20[13|15|17|19|22]/Debug

Location of the import libraries for the DLLs described above.

<InstallDir>/Source/bin/<PlatformDir>/VS20[13|15|17|19|22]/DebugUnicode

<InstallDir>/Source/lib/<PlatformDir>/VS20[13|15|17|19|22]/DebugUnicode

Location of the import libraries for the DLLs described above.

<InstallDir>/Source/bin/<PlatformDir>/VS20[13|15|17|19|22]/Release

Location of DLLs that are built with Visual Studio 20[13|15|17|19|22]. The DLLs do not contain debug information and use the multi-byte character set.

<InstallDir>/Source/lib/<PlatformDir>/VS20[13|15|17|19|22]/Release

<InstallDir>/Source/lib/<PlatformDir>/VS20[13|15|17|19|22]/ReleaseUnicode

Location of the import libraries for the DLLs described above.

 

SDK Components

The TB5CPP component of the SDK that provides the C++ API is delivered as binary. The needed header files can be found at the following locations:

Directory location

Description

<InstallDir>/Source/api/include

Location of the class header files for the C++ API.

<InstallDir>/Source/api/include/ModelCompiler

Location of the header files that define the public API of the Model Designer components.

<InstallDir>/Source/Core/include

Location of the class header files for the TB5OT component.

•CPPTestClient.exe: C++ test client application.

The following table depicts locations associated with these applications.

Description
