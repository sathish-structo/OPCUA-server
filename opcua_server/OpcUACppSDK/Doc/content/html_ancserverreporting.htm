Event Registry

Top  Previous  Next

The EventRegistry is a storage for conditions that are being reported. From the event registry an instance of a Condition or derived class can be obtained by the reported EventId.

A Condition is stored in this registry when it is first reported, and it is only removed when it is destroyed. Further reporting of the same Condition (with a different state) will create a new EventId and replace the existing EventId in the event registry.

There are two purposes for such an event registry:

•find the correct Condition instance for an EventId as input argument of a UA method (e.g. Acknowledge).
The Condition can be retrieved from the event registry via AddressSpaceRoot::getEvent
•update a client subscription with all notifications which needs attention in case of a refreshCondition request

The EventId is handled automatically in the OPC UA C++ Server & Client SDK V6.00, i.e. every time an event is reported, an unique ID is generated and set to the EventId property.

Note:

The EventId property shall not be changed after reporting. This might cause memory leaks.
