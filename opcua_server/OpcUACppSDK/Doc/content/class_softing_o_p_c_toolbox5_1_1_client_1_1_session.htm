The Session class represents a connection to a server.

In order to connect to a server, a user has to create a session object, configure it (set the server url, security parameters, etc.), and call the connect() method. Please note that sessions have to be added to the Application object (see Application::addSesion()).

Unless specified otherwise, the session's security- and connection-related attributes cannot be modified when the session is connected.

EnumStatusCode SoftingOPCToolbox5::Client::Session::configure(const tstring & discoveryUrl, const UserIdentityToken & userIdentityToken, EnumMessageSecurityMode messageSecurityMode, const tstring & securityPolicy, const tstring & userSecurityPolicy, const tstring & protocolFilter, OTUInt16 portFilter)

Configures all server related settings for this session.

Performs a GetEndpoints service, selects the desired endpoint configuration based on the desired filters and configures this session from the selected configuration.

A URL has the format "<transport>://<host or IP>[:<port>][/<url-path>]". Supported transport are "opc.tcp" and "https" Omitted port means default port 4840.

The method will

•invoke the Application::getEndpointsFromServer
•search for a matching endpoint description
•configure the session URL - it may be different to discovery URL due to filtering of protocol, host name, port, ...
•configures security related settings (server certificate, security configuration, user token security)
•configures the user identity token (with correct PolicyId from server)

Parameters:

discoveryUrl

The url of the endpoint where this session can request the configuration settings.

userIdentityToken

The information for the credentials of the user connecting to the server. The PolicyId can be left empty - it will be identified by SDK If no SecurityPolicyUri string is configured, the SDK will search for the best policy provided by the server.

messageSecurityMode

A filter for the desired message security mode the session shall use. EnumMessageSecurityMode_Invalid will select the highest supported mode.

securityPolicy

A filter for the desired message security policy. An empty policy will select for the best supported policy.

userSecurityPolicy

A filter for the desired user token security. Some user tokens can use security to encrypt the secret of the user credentials. An empty policy will select the best supported policy provided by the server.

protocolFilter

A filter to select the desired protocol for the endpoint URL. A server may expose multiple endpoints using different protocols. Endpoints for different protocols are ignored. An empty protocol string will use the protocol given in the discoveryUrl (only of no port filter is specified).

portFilter

The port part of the URL the session shall use. Endpoints using different ports are ignored. Value 0 will skip the port filtering.

Returns:

EnumStatusCode_Good A matching endpoint description was found and configured, the session is ready to be connected. EnumStatusCode_BadServerUriInvalid The discovery URI is syntactical incorrect. EnumStatusCode_BadSecurityPolicyRejected The security policy or user security policy is not supported or invalid. EnumStatusCode_BadNotFound No matching endpoint description found. EnumStatusCode_BadCommunicationError The GetEndpoints service fails to connect to the server.

Note:

This method can be invoked only if the server is accessible.

Even once the session is configured, a connect still may fail due to security issues. Even with message security None, trust relationship needs to be established if user token needs encryption. Trust relationship is needed (most likely) on both server and client!

If this method finds several matching endpoint or user token configurations, it selects a configuration with the highest security level

EnumStatusCode SoftingOPCToolbox5::Client::Session::setSessionTimeout(double sessionTimeout)

Sets the requested maximum number of milliseconds that a session should remain open without activity. If the client fails to issue a communication within this interval, then the server shall automatically terminate the session. Any further communication using this session is no longer possible until client re-connects the session.

The server can revise that timing and return the revised value to the client, which might differ from the initial value requested by the client. The revised value can be obtained by getRevisedSessionTimeout() after the connect() operation has succeeded.

The default value is 60000 (1 min.). If a different value is required it should be set before connecting the session.

Parameters:

sessionTimeout

Timeout in milliseconds

Note:

Changed timeout while session is connected will have no immediate effect. Please use disconnect() / connect() to use the new timeout.

Invokes the OPC UA read service, which reads a set of values from the connected server. The service allows to perform several read requests within one transaction. If the return code is bad the complete transaction failed and the values vector is undefined. Single request errors are reported in the values output parameter.

The read can also be called asynchronously (see readAsync()).

Note regarding maxAge parameter

In case of a ReadTransaction containing many requests the server is sometimes not able to fulfill the timing requests completely. It can happen that some Values can be read from the cached values where others have to be read from an extern device. If the read from the extern device takes a long time the already finished request might became older than the max. requested age.

Parameters:

The maximum age that is allowed for the returned Value in milliseconds. That allows the server to return a cached value for a request instead of re-reading from an external device. This parameter is used for all requests in the transaction. The value is given in milliseconds where

•in case of maxAge = maxInt32 the server is allowed to return any cached value.
•in case of 0 the server shall always read the value from the device.

A vector of returned information containing status code, timestamps and value. For each request such an information is returned. nodesToRead and values are linked by the same index in the vectors. In case of a bad request status code, the timestamp and value information is undefined.

This is an overloaded member function, provided for convenience. It differs from the above function only in what argument(s) it accepts. The relative paths used in this overload follow the following BNF notation:

<relative-path> ::= <reference-type> <browse-name> [relative-path]

<reference-type> ::= '/' | '.' | '<' ['#'] ['!'] <browse-name> '>'

<browse-name> ::= [<namespace-index> ':'] <name>

where the reference type abbreviations mean

•"/" : Hierarchical references (and sub-types)
•"." : Aggregates references (and sub-types)

The optional parts in the explicit reference type syntax mean

•"#" do not follow sub-types of specified reference type
•"!" follow references backwards

Example for equivalent relativePaths:

"/1:MyFolder.1:MyElement" == "<0:HierarchicalReferences>1:MyFolder<0:Aggregates>1:MyElement"
