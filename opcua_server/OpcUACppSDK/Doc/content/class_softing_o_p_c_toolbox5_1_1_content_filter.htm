#include "ContentFilter.h"

 

The ContentFilter data type is used by event subscriptions (see IEventFilter) and the query services (see Client::Session::queryFirst() and Client::Session::queryNext()) to specify filter criteria for events resp. nodes.

A filter consists of one or more filter elements that contain both a filter operator ('And', 'Or', 'Like', 'OfType', ...) and filter operands.

A filter is evaluated by evaluating the first entry in the element list starting with the first operand in the operand list. The operands of an element may contain references to sub-elements (see ElementOperand) resulting in the evaluation continuing to the referenced elements in the element array. If an element cannot be traced back to the starting element it is ignored.
