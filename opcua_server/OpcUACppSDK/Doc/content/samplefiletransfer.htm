File Transfer

Top  Previous  Next

This sample shows how a server can expose individual files using the FileType ObjectType and how a client can handle these files using the properties and methods of this type.

 

Where to Find

The sample projects are located in the directory: "<InstallDir>/Apps/Samples/Cpp/FileTransfer" and the sources are in the subdirectories "FileTransferServer" and "FileTransferClient".

The actual files exposed by the server sample are in the "ServerFiles" subdirectory.

 

Server Implementation

The server creates and exposes two FileType nodes organized under a "ServerFiles" Folder object. The File class instances are initialized with the paths to the physical files and the Writable property is set to make first file writable and the second read-only.

 

Running the Sample

•The server needs an existing application instance certificate and it must be in the correct directory. See Setting up a Certificate Store for how to create the demo certificate store. The path to the certificate is specified at the top of FileTransferServer.cpp, adapt it if it doesn't fit.
•Compile and start the sample application FileTransferServer. The server is listening for incoming connections at the address "opc.tcp://localhost:4880".
•Connect to the sample server with your favorite OPC UA client (e.g. Softing OPC UA Client), browse the folder "ServerFiles" and check the properties of the two FileType nodes (ReadOnlyFile and ReadWriteFile).
•The FileTransferClient application provides a simple interface to call the methods of the File objects.

 

Points of Interest

•How to represent real-world Files in the address space of a server by using the File class (see function createAddressSpace() in FileTransferServer.cpp).
•Use the Writable property to restrict changes on a server file.

 

Client Implementation

The client application shows basic operations on the FileType nodes exposed by the server. It is hard coded to browse for FileType nodes under the "ServerFiles" Folder object.

For convenience the helper class ClientFile was created. It wraps all low level calls performed on the Client::Session object, providing a good separation between the actual SDK calls and the code written for user interaction.

For method calls the well known NodeId defined by FileType is used while for property reads the NodeIds are found and cached by the getFileComponentNodes() method on first call.

 

Running the Sample

•The client expects the server to run at "opc.tcp://localhost:4880", if this does not match, adapt the global variable g_serverUrl at the top of FileTransferClient.cpp.
•Compile and start the sample applications FileTransferServer and FileTransferClient.
•The available commands are listed by the brief usage message on startup.
•Use the client command "co" to connect to the server.
•Use the client command "di" to disconnect from the server. Any currently open file will be closed.
•Use the client command "ls" to list all available FileType nodes. In addition to finding the references the values of the properties are displayed.
•Use the client command "op" to open a file. You will be prompted to select between available FileType nodes and to select an open mode.

The rest of the available commands are used with a previously open file, you will have to choose from a list:

•"cl" - Close a file.
•"gp" - Get file's current position.
•"sp" - Set file's current position.
•"rd" - Read from file's current position. It will display file size and position, you have to specify the amount to read (in bytes).
•"wr" - Write data to file's current position.

 

Note:

The sample is a minimal implementation and is not aware of extreme situations. Using "wr" (write) command with a big amount of data might cause the client to become unresponsive.
