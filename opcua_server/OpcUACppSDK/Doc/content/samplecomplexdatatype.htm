ComplexDataType

Top  Previous  Next

This sample shows how to define and use custom data types like structures, enumerations and unions in a UA application.

 

Where to Find

The sample projects are located in the directory: "<InstallDir>/Apps/Samples/Cpp/ComplexDataType" and the sources are in the subdirectories "ComplexDataTypeServer" and "ComplexDataTypeClient".

 

Server Implementation

The server creates and registers a DictionaryVariable as a container of custom defined data types.

For each defined data type,

•a data type description (StructuredDataTypeDescription or EnumerationDataTypeDescription) has to be defined
•a data type node has to be created
•the data type description has to be assigned to the data type node

The server sample creates the following custom data types

•a Structure
•an Enumeration
•a Union

For each custom data type the server exposes two variables, one variable for a static value and one variable where the value changes automatically.
The automatic changing variables make it easy to monitor value changes.
All variables are directly exposed below the "Objects" folder.

The server shows how the DataTypeDescriptions can be easily used to get and set values and encapsulates the entire initialization and value access into specific classes (see MyStructType, MyEnumerationType and MyUnionType).

Note: The value access is similar on client and server.

 

Running the Sample

•The server needs an existing application instance certificate and it must be in the correct directory. See Setting up a Certificate Store for how to create the demo certificate store. The path to the certificate is specified at the top of ComplexDataTypeServer.cpp, adapt it if it doesn't fit.
•Compile and start the sample application ComplexDataTypeServer. The server is listening for incoming connections at the address "opc.tcp://localhost:4880".
•Connect to the sample server with your favorite OPC UA client, that is capable to handle custom data types (e.g. Softing OPC UA Client) and browse and access the variables with the complex data type values.
oBrowse the folder Objects and read, write or subscribe for data changes on any variable below.
oYou can also use the ComplexDataTypeClient to test the servers functionality.

 

Points of Interest

•How to create the dictionary variable as a container for all custom data types by use of DictionaryVariable (see function createAddressSpace() in ComplexDataTypeServer.cpp).
•How to setup the new custom UA data types by the use of the DataType, DataTypeDescription and StructureField or EnumField (see MyStructType::initialize(), MyEnumerationType::initialize()and MyUnionType::initialize()).
•How to convert a structure from and to a Value instance (see MyStructType::setMyStruct() and MyStructType::getMyStruct()).
•How to convert an enumeration from and to a Value instance (see MyEnumerationType::setMyEnumerationValue() and MyEnumerationType::getMyEnumerationValue())
•How to convert a union from and to a Value instance (see MyUnionType::setMyUnion() and MyUnionType::getMyUnion())

Note: The get methods (MyStructType::getMyStruct(), MyEnumerationType::getMyEnumerationValue() and MyUnionType::getMyUnion()) are currently not called because the sample sets only values to the address space while the read or subscribe access is completely handled by SDK.
These getters make sense in case you access the received values from the client (e.g. a custom write handling or method call handling) or if you re-access the values which are already stored in the server address space.

 

Client Implementation

The client retrieves the DataTypeDescriptions, which are defined on the server and validates that the types are as expected.

It shows how the DataTypeDescriptions can be easily used to get and set values and encapsulates the entire initialization and value access into specific classes (see MyStructType, MyEnumerationType) and MyUnionType).

Note: The value access is similar on client and server.

The client demonstrates the custom data type usage by reading and writing the variables of the ComplexDataTypeServer.

 

Running the Sample

•The client expects the server to run at "opc.tcp://localhost:4880", if this does not match, adapt the global variable g_serverUrl at the top of ComplexDataTypeClient.cpp.
•Compile and start the sample applications ComplexDataTypeClient and ComplexDataTypeServer.
•Use the client command "co" to connect to the server, this also retrieves and validates the used DataTypeDescriptions.
•Use the client command "rd" to read the static and dynamically changing variables with the structure, enumeration or and union data types.
•Use the client command "wr" to write the static variables for with some generated structure, enumeration and union values.
