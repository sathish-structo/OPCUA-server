#include "ServerWriteTransaction.h"

 

The WriteTransaction allows server applications to process 'Write' service calls. Clients use the 'Write' service to write attribute values of one or more nodes on the server. The transaction contains a WriteRequest object for each node and attribute value that is to be written.

The SDK creates a new transaction object for every service call and triggers the handleRequests() callback that is responsible for processing the call. The default implementation of handleRequests() calls BaseNode::handleWriteRequest() for each node and attribute that is to be read.

A transaction is completed when all requests in the transaction have been completed by WriteRequest::complete(). Only when the transaction is completed, the response to the service call is sent to the client.

virtual EnumStatusCode SoftingOPCToolbox5::Server::WriteTransaction::handleRequests() [virtual]

Callback method that is called by the SDK whenever a client uses the 'Write' service to write attribute values of one or more nodes.

The default implementation calls BaseNode::handleWriteRequest() for every node and attribute that is to be written.

Note:

When overriding this method you mustn't use blocking calls within this method since this blocks the UA communication.

The requests within this transaction are already preprocessed by the SDK. If the SDK has found any errors (e.g. EnumStatusCode_BadNodeIdUnknown, EnumStatusCode_BadNotWritable...) in most cases the request should not be processed any more. Just complete the request and continue with the next. In some cases (e.g. processing "virtual" node, not known to the AddresSpaceRoot) further processing can be executed and the status code can be overwritten.
