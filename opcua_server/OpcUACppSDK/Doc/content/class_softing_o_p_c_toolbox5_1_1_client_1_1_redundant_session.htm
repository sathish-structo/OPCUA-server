#include "ClientRedundantSession.h"

 

The RedundantSession class represents a connection to an array of redundant servers.

One of the servers acts as primary server, the other servers act as secondary (back-up) servers. The role of each server in an array of redundant servers is determined by the "ServiceLevel" variable which each server exposes in its address space. Each "ServiceLevel" variable contains a value ranging from 0 to 255. The server with the highest service level has the role of the primary server. The actual service level of all servers is reported by RedundantSession::onServiceLevelChanged().

Upon a failure of the primary server, the client may perform a switchover to a secondary server (see RedundantSession::switchoverAsync()). From the perspective of the client, the switchover changes the role of the designated server from 'secondary' to 'primary'.

In order to connect to an array of redundant servers, the application has to create a RedundantSession object, configure it by setting the server URLs, security parameters, etc., and call the connect() method. Please note that redundant session need to be registered at the Application object (see Application::addRedundantSession()).

Note:

Unless specified otherwise, the session's security- and connection-related attributes cannot be modified while the session is connected.

EnumStatusCode SoftingOPCToolbox5::Client::RedundantSession::setSessionTimeout(double sessionTimeout)

Sets the requested maximum number of milliseconds that a session should remain open without activity. If the client fails to issue a communication within this interval, then the server shall automatically terminate the session. Any further communication using this session is no longer possible until client re-connects the session.

The server can revise that timing and return the revised value to the client, which might differ from the initial value requested by the client. The revised value can be obtained by getRevisedSessionTimeout() after the connect() operation has succeeded.

The default value is 60000 (1 min.). If a different value is required it must be set before connecting the session.

Parameters:

sessionTimeout

Timeout in milliseconds

Note:

Changed timeout while session is connected will have no immediate effect. Please use disconnect() / connect() to use the new timeout.

EnumStatusCode SoftingOPCToolbox5::Client::RedundantSession::callMethodsAsync(void * transactionHandle, const std::vector< CallMethodRequest > & methodCallRequest, OTUInt32 transactionId, OTUInt32 redundantServerIndex)

Invokes the OPC UA method call service asynchronously. The service allows to perform several method call requests within one transaction. The call returns immediately after sending the service to the server. When the response is available, the callback method onCallMethodsComplete() is invoked.

The method call can also be called synchronously (see callMethods()).

Parameters:

transactionHandle

A cookie specified by the caller. This cookie can be used to identify the service in the asynchronous response.

methodCallRequest

A vector of information needed for calling a method. For detailed information see CallMethodRequest

transactionId

A number to group a set of transactions. The ID needs not to be unique and can be any number except 0xFFFFFFFF. The ID can be used to cancel transactions. In case of multiple transactions sharing the same ID all transactions with this ID can be canceled using one call (see cancelTransaction()).

redundantServerIndex

The index of the redundant server where the service is called on.

Returns:

The status code of the service sent to the server. In case of bad service status code the complete transaction failed and no callback will be invoked.

See also:
