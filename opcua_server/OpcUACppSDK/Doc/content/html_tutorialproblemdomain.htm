The sample applications that are developed in this tutorial cover a domain that most people will be familiar with: kitchen appliances, or more specifically, refrigerators. The server tutorial illustrates the implementation of an OPC UA server that exposes the operating conditions of several refrigerators and provides means to influence those conditions. The client tutorial describes the implementation of a text-based OPC UA client that allows you to monitor and control the refrigerators exposed by the server.

The refrigerators (all of them are the same model) expose several data points via a proprietary interface:

•Actual Temperature (read-only): The current temperature inside the refrigerator
•Temperature Setpoint (read/write): The desired temperature inside the refrigerator
•Light Status (read-only): Indicates if the light in the refrigerator is on
•Door Closed (read-only): Indicates if the door of the refrigerator is closed
•Motor Temperature (read-only): The temperature of the refrigerator's cooling motor

The server application that is developed in this tutorial accesses the refrigerators via their proprietary interfaces and expose all information in an OPC UA address space. The client application accesses the information exposed by the server and is able to monitor and control the refrigerators.
