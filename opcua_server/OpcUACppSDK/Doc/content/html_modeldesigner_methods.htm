Introduction

Every invocation of a model designer method updates the "Last Action" variable to expose the latest address space changes. Failed attempts to modify the address space are reported as well.
The variable can be used to list all changes in the address space by subscribing to it.

 

Model Designer Methods

There are now the following methods available:

•AddNamespace
•ModifyNamespace
•CreateInstance
•CreateMethod
•CreateObjectType
•CreateVariableType
•CreateReferenceType
•ModifyAttributes
•GetExtensions
•SetExtensions
•AddReference
•RemoveReference
•RemoveNode
•MoveNode
•Export
•Import
•ResetAddressSpace

In some cases it is possible that a method is rejected due to restrictions that would lead to an invalid address space. E.g. the RemoveNode method is called to remove a type with existing instances of that type., or if the CreateMethod is configured to add the method on something else as an Object or ObjectType. If a method is rejected, the reason is well documented in the Result output.
