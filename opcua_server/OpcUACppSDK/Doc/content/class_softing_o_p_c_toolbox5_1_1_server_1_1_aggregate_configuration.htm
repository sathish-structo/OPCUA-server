bool SoftingOPCToolbox5::Server::AggregateConfiguration::getTreatUncertainAsBad() const

Returns whether values with 'Uncertain' status code are considered equally as values with 'Bad' status code when calculating an aggregate value.

See also:

setTreatUncertainAsBad()

EnumStatusCode SoftingOPCToolbox5::Server::AggregateConfiguration::setTreatUncertainAsBad(bool treatUncertainAsBad)

Sets whether values with 'Uncertain' status code are considered equally as values with 'Bad' status code when calculating aggregate values.

Parameters:

treatUncertainAsBad

If true, 'Uncertain' values are considered as 'Bad' values when calculating aggregate values, i.e. 'Uncertain' values are not included in the aggregate calculation.

Note:

Default value is true.

OTUInt8 SoftingOPCToolbox5::Server::AggregateConfiguration::getPercentDataBad() const

Returns the minimum percentage of 'Bad' values in a processing interval required for status code of the interval to be set to 'Bad'.

See also:

setPercentDataBad()

EnumStatusCode SoftingOPCToolbox5::Server::AggregateConfiguration::setPercentDataBad(OTUInt8 percentDataBad)

Sets the minimum percentage of 'Bad' values in a processing interval required for status code of the interval to be set to 'Bad'.

Please note that the following relationship must hold true for PercentDataGood and PercentDataBad: PercentDataGood >= (100 - PercentDataBad). If both sides of the inequation are equal, the result of the PercentDataGood calculation is used.

Parameters:

percentDataBad

The minimum percentage of 'Bad' values required for the status code of the processing interval to become 'Bad'.

Note:

Default value is 100.

bool SoftingOPCToolbox5::Server::AggregateConfiguration::getUseSlopedExtrapolation() const

Returns whether the server uses linear extrapolation (sloped extrapolation) or extrapolation based on horizontal line fit (stepped extrapolation) for projecting values that are farther in the future than available in the historian.

See also:

setUseSlopedExtrapolation()

EnumStatusCode SoftingOPCToolbox5::Server::AggregateConfiguration::setUseSlopedExtrapolation(bool useSlopedExtrapolation)

Defines whether the server uses linear extrapolation (sloped extrapolation) or extrapolation based on horizontal line fit (stepped extrapolation) for projecting values that are farther in the future than available in the historian.

Parameters:

useSlopedExtrapolation

If true, the server uses linear extrapolation for projecting values. If false, the server uses horizontal line fit (i.e. the last non-bad value is assumed as constant and used as projected value).

Note:

Default value is false.
