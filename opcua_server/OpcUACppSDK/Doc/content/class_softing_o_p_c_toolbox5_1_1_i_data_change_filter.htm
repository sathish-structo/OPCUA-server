#include "IDataChangeFilter.h"

 

The DataChangeFilter data type is used by data change subscriptions (see Client::MonitoredItem::setFilter()) to specify the conditions under which a data change notification should be reported and, optionally, a range or band for value changes where no data change notification is generated. This range is called "deadband".

virtual void SoftingOPCToolbox5::IDataChangeFilter::setTrigger(EnumDataChangeTrigger value) [pure virtual]

Sets the conditions under which a data change notification should be reported.

The following triggering options are supported:

•Report data change notifications only if the status code associated with the value changes (see EnumDataChangeTrigger_Status)
•Report data change notifications if either the status code or the value changes (see EnumDataChangeTrigger_StatusValue)
•Report data change notifications if either the status code, the value or the timestamp changes (see EnumDataChangeTrigger_StatusValueTimestamp)

The following types of deadbands are supported:
