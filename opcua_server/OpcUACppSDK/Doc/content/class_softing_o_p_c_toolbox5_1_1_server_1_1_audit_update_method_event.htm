#include "ServerAuditUpdateMethodEvent.h"

 

Base class of all method related audit events.

The SourceNode property for events of this type should be assigned to the NodeId of the object that the method resides on. The SourceName property for events of this type should be "Attribute/Call".

Beside mandatory properties of the base class additionally the following mandatory properties shall be set:

•MethodId: Identifies the method that was called.
•InputArguments: Identifies the input arguments for the method. This property can be omitted if no input arguments have been provided.

void SoftingOPCToolbox5::Server::AuditUpdateMethodEvent::setMethodAuditInformation(const Server::MethodCallRequest * request, const Server::MethodCallTransaction * transaction, bool status)

Sets the method related information.

The properties filled with method are:

•MethodId: Identifies the method that was called.
•InputArguments: Identifies the input Arguments for the method.
•ClientAuditEntryId: Contains the human-readable AuditEntryId.
•SourceNode: Identifies the Node that the Event originated from.
•Status: Identifies whether the requested action could be performed.

Parameters:

request

The request of the method call

transaction

The transaction containing the request

status

The Status property identifies whether the requested action could be performed (set Status to TRUE) or not (set Status to FALSE).

EnumStatusCode SoftingOPCToolbox5::Server::AuditUpdateMethodEvent::setComment(ILocalizedText * comment)

Sets the comment property of an audit event.

The comment property is mandatory for the following derived audit event types:

•AuditConditionCommentEvent: Report the "AddComment" action.
•AuditConditionAcknowledgeEvent: Indicates acknowledgment or confirmation of one or more conditions.
•AuditConditionConfirmEvent: Reports a "Confirm" action.

All other derived audit events shall not set the comment property.
