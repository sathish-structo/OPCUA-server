Setting up a Project for VxWorks

The following instructions describe how to set up a Wind River Workbench project to use the OPC UA C++ Server & Client SDK with VxWorks.

Step 1

Please start the Wind River Workbench.

Step 2

•Select "New"->"Project" in the context menu of the "Project Explorer"
•Create a "Wind River Workbench Project"
•Target operating system: Wind River 6.x (select your version of VxWorks here)
•Build type: Real Time Process Application
•Project name: Name of your application
•Location: Create project in workspace
•Finish

Step 3

Please open the properties of the project and select "Build Properties". Add at least one C++ source file to the project if this options is not available. Then, make the following settings:

•Tab "Tools"
oBuild Tool: "C++-Compiler"
▪add "-fno-strict-aliasing" to the Tool Flags (GNU GCC only)
•Tab "Paths"
oPlease ensure that the following paths are present in the list (they should be defined by default):
▪-I$(WIND_BASE)/target/usr/h
▪-I$(WIND_BASE)/target/usr/h/wrn/coreip
oAdd the following paths using the "Add to all..." button:
▪-I<InstallDir>/Source/api/include
▪-I<InstallDir>/Source/Core/include
•Tab "Libraries"

Apply the following settings using the "Add to all..." button:
Note: Replace whole string, not just highlighted part!

▪-L <InstallDir>/Source/lib/VxWorks<version>/<Arch>/NonDebug
▪-l tb5cpp
▪-l tb5ot
▪-l xml2
▪-l tb5util
▪-l tb5stack
▪-l crypto

Step 4

Build the project (do not click on "Generate Includes...").

Step 5

In order to run the test applications on a VxWorks target system, the kernel of the systems needs to be configured with certain components.
Please ensure that the following components are included in your image:

•INCLUDE_RTP
•INCLUDE_POSIX_PTHREAD_SCHEDULER

Note: Typically the easiest approach to find these component settings is to use the search function (Ctrl + f) in the Kernel Configuration view and search for the described names.

Next Steps

You are now ready to develop an OPC UA-enabled application using the OPC UA C++ Server & Client SDK. Please ensure to always include the Toolbox.h header file as your first SDK-related header file when using the SDK in one of your compilation units.

If you are new to the SDK, you may want to read the introduction to the C++ API and work through the Programming Tutorial.
