This topic belongs to Get NodeIds by Browse Paths and Get NodeIds Using the Namespace Table. It refers to the sample application Client_5 and should be tested with the sample application Server_4. (See Programming Tutorial for where to find the tutorial samples)

Where to Find

Source code and project files belonging to the sample application are located in the directory <InstallDir>/Apps/Samples/Cpp/Tutorial/Client_5.

Running the Client

•Compile and start the sample applications Client_5 and Server_4. Run the client application in the debugger and step through the code.
•Connect to the server with the command "co" and get the NodeIds of the variables, instances and components by calling a browse with "br".
•Use the command "rd" to read the values of the refrigerator variables and "wr" to write the setpoint temperature. The commands have additional optional parameters to specify the affected refrigerator instance. The "wr" command also has a parameter to specifiy the desired setpoint value.
•Use the command "sub" (subscribe) to subscribe to value changes of the actual temperature and "stp" (stop) to stop the subscription. These commands also have an optional parameter to specify the refrigerator instance.

Points of Interest

•The relevant code can be found in Refrigerator::getTypeNodeId(), Refrigerator::iniFromServer(), Refrigerator::readVariables(), Refrigerator::writeSetpoint(), Refrigerator::startMonitoringActualTemperature() and Refrigerator::stopMonitoringActualTemperature() in the file Refrigerator.cpp.
•The namespace table of the Session to get the namespace index of a namespace URI (See Refrigerator::getTypeNodeId())
•Use of the TranslateBrowsePathToNodeId service to get NodeIds of components of custom objects. This requires knowledge about the structure of the custom object type. (See Refrigerator::iniFromServer())
•How to read and write on a device (here: the custom refrigerator objects). (See Refrigerator::readVariables() and Refrigerator::writeSetpoint())
•How to add and remove multiple monitored items to a single subscription. When we monitor the actual temperatures of multiple refrigerators, we will only use one subscription. (See Refrigerator::startMonitoringActualTemperature() and Refrigerator::stopMonitoringActualTemperature())
