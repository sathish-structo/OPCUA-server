OPC UA C++ Server & Client SDK V6.00

Verifying the Installation

Top  Previous  Next

The SDK is delivered with source code of command line applications that may be used to verify the installation of the SDK. The testing environment comprises a Test Server and Test Client application. The Test Server application uses the SDK to implement a feature-rich OPC UA server. The Test Client application uses the SDK to implement an OPC UA client that can be controlled by script files. The SDK installation includes test script files that perform tests to verify the installation of the SDK.

Wind River projects for the test applications are delivered in the directory Workspace_x[.y].

Compiling the Test Client and Test Server Application

Please follow the steps below to compile the test applications using Wind River Workbench:

Step 1

For VxWorks 6.x this step can be omitted.

For VxWorks 7 a Wind River Source Build (VSB) project is necessary.
Please create such a project in the workspace, based on your BSP or CPU.

Note: For VxWorks 7 currently only Windows simulator binaries are delivered in binary setup.

Step 2

To import or create the test projects into the delivered workspace, please open a command shell and type the following commands:

On Microsoft Windows host systems:

> cd <InstallDir>/Workspace_x[.y]

> AddTestProjects.bat <Path to the Wind River Workbench installation>

On Linux host systems:

> cd <InstallDir>/Workspace_x[.y]

> ./AddTestProjects.sh <Path to the Wind River Workbench installation>

If you are unsure about the path to the Wind River Workbench, please open a VxWorks Development Shell and examine the content of the WIND_HOME environment variable.

Step 3

Please start the Wind River Workbench and open the workspace.

Adapt the files "StructoAutomoldServer.cpp, TestClient.cpp and CommandLineOptions.cpp: All occurrences of "tgtsvr" shall be adapted to a position, where

•PKI store can be accessed
•Source Folder of OPC UA C++ Server & Client SDK V6.00 can be accessed - files in TestFiles below StructoAutomoldServer and TestClient are needed for tests.

For VxWorks 6.x specify the desired architecture, compiler and debug mode as the active build spec.

Build the projects StructoAutomoldServer and TestClient.

After the compilation has finished, the executables should have been generated in the directory <InstallDir>/Workspace_x[.y]/<application>/<build spec>/<application>/<debug mode>

 

Preparing an Image Project

In order to run the test applications on a VxWorks target system, the kernel of the systems needs to be configured with certain components.
Please ensure that the following components are included in your image:

•INCLUDE_RTP
•INCLUDE_POSIX_PTHREAD_SCHEDULER (application will not launch if this is specified)
•Increase NUM_FILES to minimum 50 (some tests will fail if too few files are available)
•Increase RTP_FD_NUM_MAX to minimum 100 (some tests will fail if too few files are available)

Note: Typically the easiest approach to find these component settings is to use the search function (Ctrl + f) in the Kernel Configuration view and search for the described names.

 

Setting up the Certificate Store

The Test Server and Test Client applications verify that messages can be exchanged securely. Therefore, certificates with corresponding private key need to be generated for both applications. The SDK is shipped with shell scripts that creates certificate stores containing the required certificates for all applications.

In order to create certificate stores at the location <InstallDir>/Source/PKI, please open a command shell and run the following shell script:

On Microsoft Windows host systems:

> cd <InstallDir>/Source/PKI

> create_demo_store.bat

On Linux host systems:

> cd <InstallDir>/Source/PKI

> ./create_demo_store.sh

For further details see Setting up a Certificate Store
