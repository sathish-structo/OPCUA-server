Message Sending

The following image shows the steps performed by a Publisher to encode and send NetworkMessages. The entire process is guided by parameters configured for each logical step.

Step 1 - Collection of data to be published. Values for the individual fields of a DataSet are produced.

Step 2 - A DataSetWriter encodes a DataSetMessage from the collected DataSet.

Step 3 - A NetworkMessage is created. This step is performed on WriterGroup level since all DataSetMessages from DataSetWriters in one WriterGroup can be inserted into a single NetworkMessage.

Step 4 - NetworkMessage is delivered to the Message Oriented Middleware.

 

Message Reception

Subscribers are the consumers of NetworkMessages. The following image shows the simplified steps performed by a Subscriber to receive, decode and dispatch the data.

As for the Publisher, various parameters are configured for each of the main logical steps:

Step 1 - Establish a connection.

Step 2 - Receive and filter the messages based on header information.

Step 3 - A DataSetReader decodes a DataSet from a DataSetMessage.

Step 4 - Dispatching of the decoded DataSet (a SubscribedDataSet can be a mapping of DataSet fields to Variables in the Subscriber AddressSpace).
