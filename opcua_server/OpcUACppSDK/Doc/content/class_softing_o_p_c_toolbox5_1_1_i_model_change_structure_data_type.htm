IModelChangeStructureDataType Class Reference

#include "IModelChangeStructureDataType.h"

 

The ModelChangeStructureDataType contains elements that describe changes (added or removed nodes and references) in a server's address space. The data type is primarily used by GeneralModelChangeEvents.

virtual OTUInt8 SoftingOPCToolbox5::IModelChangeStructureDataType::getVerb() const [pure virtual]

Returns a bit-field that describes the changes to the affected node.

•Bit 0: If set, indicates that the affected node has been added
•Bit 1: If set, indicates that the affected node has been deleted
•Bit 2: If set, indicates that a reference has been added. The affected node may be either the source or target of a reference. Note that an added bidirectional reference is reflected by two ModelChangeStructure entries.
•Bit 3: If set, indicates that a reference has been removed. The affected node may be either the source or target of a reference. Note that an added bidirectional reference is reflected by two ModelChangeStructure entries.
•Bit 4: If set, indicates that the DataType attribute of a Variable or VariableType has changed.
•Bit 5-7: Reserved for future use. Shall always be zero.

A verb may identify several changes on the affected node at once. Note that all verbs shall always be considered in the context where the ModelChangeStructureDataType is used. A set bit 1 may indicate that a node was removed from a view but still exists in other views.
