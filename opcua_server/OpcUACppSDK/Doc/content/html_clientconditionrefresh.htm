Receive All Current Alarms

Top  Previous  Next

OPC UA provides means to retrieve the current status of all monitored alarms.

The method conditionRefresh on a subscription is exactly performing this task. Once a refresh is requested, the subscription will receive the following event notifications for every monitored item:

1.A RefreshStartEvent is triggered with the source node set to the matching monitored item
The RefreshStartEvent is a specific event type. It can be detected by comparing the EventType property (usually this property is part of the selected properties).
2.All conditions which still need attention (i.e. Retain property set to true) are reported (again). The condition will keep its EventId during this report.
If the monitored item is in the notification chain (see EventPropagation) the client will receive the notification.
3.A RefreshEndEvent is triggered with the source node set to the matching monitored item.
The RefreshEndEvent is a specific event type. It can be detected by comparing the EventType property (usually this property is part of the selected properties).

This sequence is repeated for every monitored item in the subscription.

If the client performs a conditionRefresh while another refresh is still active it will get an error code.

 

Situations when to Refresh

Usually an application will call conditionRefresh on subscriptions containing event monitored items when

•the subscription is connected initially to retrieve the current states
•the subscription is re-connected after a connection break
•the subscription receives a "RefershRequiredEvent", which is used by the server to notify the situation that some notifications are lost
The RefershRequiredEvent is a specific event type. It can be detected by comparing the EventType property (usually this property is part of the selected properties).

The RefershRequiredEvent is typically reported by the server when the queue in a monitored item detects an overflow and removed notifications not yet reported.

Note:

It is also possible that during a refresh a server issues new event notifications. That means that the notifications for one single condition or alarm appear not in time order.
