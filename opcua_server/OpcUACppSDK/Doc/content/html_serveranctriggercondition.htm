Alarm Reporting

Top  Previous  Next

An alarm usually monitors an input variable and evaluates a condition. The different alarm types (e.g. ExclusiveLimitAlarm, NonExclusiveLevelAlarm, ...) handle the evaluation in a different way.

The default implementation of the predefined alarm types are implemented and work on the precondition that changed input values are updated in the variable node.

In case of special handling (e.g. implementation of condition branches, ...) the application can overwrite the onVariableChanged method in a derived class.
Ensure that the major functionality is implemented in the overwritten method:

•set the Quality property(default uses quality of input variable)
•evaluate condition (different implementations on specialized types)
•if a state is changing
•update the current state(s)
•set the ActiveState property to the new active state (might be the same, just a different sub-state)
•set Time and ReceiveTime property
•call onActivate / onDeactivate depending on new active state

The other properties (e.g. Retain, Acknowledged, ...) will be set automatically.
