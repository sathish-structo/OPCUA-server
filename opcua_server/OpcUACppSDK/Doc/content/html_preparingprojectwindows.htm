Setting up a Project on Windows

The following instructions describe how to use the OPC UA C++ Server & Client SDK in a Visual Studio development environment.

Step 1

Please open Visual Studio and create a new/open an existing Visual C++ project.

Step 2

Open the project properties of your application and add the following include directories (Configuration Properties -> C/C++ -> General: Additional Include Directories):

•<InstallDir>/Source/api/include
•<InstallDir>/Source/Core/include

Step 3

In the project properties, add the following library directory (Configuration Properties -> Linker -> General: Additional Library Directories):

•<InstallDir>/Source/lib/<PlatformDir>/<Visual Studio Version>/<Build Configuration>

The library directory you choose here depends on the version of your Visual Studio installation and your build configuration. The Installation Directory Structure lists and describes all available build configurations. Please note that the selected build configuration should match the build configuration of your application.

Hint: If you are unsure about the character set to choose, please look up the character set that your application uses in the project properties (Configuration Properties -> General: Character Set).

Step 4

Again in the project properties, add the import library TB5CPP.lib for 32 bit library or TB5CPPx64.lib for 64 bit library to the library dependencies of your application (Configuration Properties -> Linker -> Input: Additional Dependencies).

Step 5

Since the SDK libraries are linked dynamically, the operating system must be able to find the required SDK DLLs at runtime. One option is to copy the SDK DLLs to the same directory in which your executable resides. The following DLLs must be available at runtime:

For a 32 bit application:

•TB5CPP.dll
•TB5OT.dll
•TB5STACK.dll
•TB5UTIL.dll
•libcrypto-1_1.dll
•libssl-1_1.dll
•libxml2.dll

For a 64 bit application:

•TB5CPPx64.dll
•TB5OTx64.dll
•TB5STACKx64.dll
•TB5UTILx64.dll
•libcrypto-1_1.dll
•libssl-1_1.dll
•libxml2x64.dll

Next Steps

You are now ready to develop an OPC UA-enabled application using the OPC UA C++ Server & Client SDK. Please ensure to always include the Toolbox.h header file as your first SDK-related header file when using the SDK in one of your compilation units.

If you are new to the SDK, you may want to read the introduction to the C++ API and work through the Programming Tutorial.
