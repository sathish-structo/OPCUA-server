Address Space Modeling

Top  Previous  Next

The Address Space of an OPC UA Server represents a coherent collection of meta information that enables a client to identify in an intuitive manner the interesting data-sources within the OPC UA server. The OPC UA Address Space consists of a collection of Nodes, uniquely identified by NodeId and linked one to each-other via References, resulting a graph that can be exposed by the server via browsing services. The OPC UA Specification defines an extensive set of rules of creating the address space in order to ensure coherence and compliance of various OPC UA Servers, and the clients to be able to predictably visualize the information. Furthermore the OPC UA Standard defines an XML based schema that can be used to define and validate in a generic and standardized manner OPC UA Server's address spaces.

OPC UA C++ Server & Client SDK features the option that a server can dynamically shape its address space by loading NodeSet2 schema configuration xml files. This way, a server's configuration can be now flexible and interchangeable. The developer of an OPC UA Server will now only have to focus on linking in a generic manner the address space components to the real life I/O sources, and not bother about generation of the various nodes and references required. Just based on external standard xml configuration files, a server can change it's shape, reducing tremendously the commissioning and deployment efforts in real life installations.

For modeling the address space, e.g. add or remove nodes and references the SDK exposes methods for these actions described in ModelDesigner Methods.

The methods are available via the AddressSpaceRoot::enableModeling where a folder for model designer containing all methods is added.

Technical details on how to put together the bits and pieces can be found in the following article: NodeSet import and validation.
