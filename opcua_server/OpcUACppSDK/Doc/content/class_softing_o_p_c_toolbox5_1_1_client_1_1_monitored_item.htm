MonitoredItem Class Reference

#include "ClientMonitoredItem.h"

 

The MonitoredItem class is used to monitor data changes or events in a server.

For monitoring data changes, the monitored item needs to be configured with the NodeId of the monitored node and the attribute of the node that is to be monitored (see setNodeId() and setAttributeId()). Usually, this will be the value attribute of a variable. The sampling interval determines the rate that the server uses to update the monitored attribute (setSamplingInterval()).

For monitoring events, the monitored item needs to be configured with the NodeId of an object that emits events and the special attribute 'EnumAttributeId_EventNotifier' (see setNodeId() and setAttributeId()). An event filter specifies both the fields of an event that the client is interested in and the criteria for an event to be reported to the client (see setEventFilter()).

A configured monitored item is created on the server only after it has been added to a subscription and connected (see Subscription::addMonitoredItem() and connect()).

EnumStatusCode SoftingOPCToolbox5::Client::MonitoredItem::addItemsToReport(const std::vector< Client::MonitoredItemPtr > & itemsToReport)

Adds a list of MonitoredItems that shall be reported when this MonitoredItem triggers.

OPC UA allows setting up MonitoredItems that report their data changes or events only if they are triggered by some other MonitoredItem. The triggering mechanism is a useful feature that allows clients to reduce the data volume on the wire by configuring some MonitoredItems to sample frequently but only report when some other event happens.

The monitoring mode of the items to report (i.e. triggered items) is set to sampling-only so that it will sample and queue notifications without reporting them.

The following triggering modes are defined:

•If the monitoring mode of the triggering item is SAMPLING, then it is not reported when the triggering item triggers the items to report.
•If the monitoring mode of the triggering item is REPORTING, then it is reported when the triggering item triggers the items to report.
•If the monitoring mode of the triggering item is DISABLED, then the triggering item does not trigger the items to report.
•If the monitoring mode of the item to report is SAMPLING, then it is reported when the triggering item triggers the items to report.
•If the monitoring mode of the item to report is REPORTING, this effectively causes the triggering item to be ignored. All notifications of the items to report are sent after the publishing interval expires.
•If the monitoring mode of the item to report is DISABLED, then there will be no sampling of the item to report and therefore no notifications to report.

Parameters:

itemsToReport

A list of MonitoredItems that shall report their notifications when this MonitoredItem triggers.

Note:

Please note that both the reporting MonitoredItems and the triggering MonitoredItem need to be added to a subscription before the triggering linkage can be established. The triggered monitored items must not be connected after the triggering monitored item (connecting in the same call works).

EnumStatusCode SoftingOPCToolbox5::Client::MonitoredItem::setIndexRange(const INumericRange * pIndexRange)

Sets the index range that the server applies to DataValues that contain variants of type 'array' or 'matrix'. The index range allows clients to request only a certain range of an array.

Examples:

•"1" - would select the second array element only
•"1:2" - would select the second and third elements in an array
•"1:2,0:1" - would select an range of a matrix.
•"1,2" - would select the element at position [1,2] in the matrix

Parameters:

pIndexRange

The index range to apply to arrays

Note:

The index range cannot be modified after the MonitoredItem has been connected.

The second argument of the range specification for a dimension must be greater than the first one. The following values are, for example, invalid: "1:1", "2:1"

EnumStatusCode SoftingOPCToolbox5::Client::MonitoredItem::setMonitoringMode(EnumMonitoringMode monitoringMode)

Sets the monitoring mode of the MonitoredItem. The monitoring mode determines whether the MonitoredItem collects notification messages (i.e. data change notifications or event notifications) and, if so, whether the Subscription publishes the collected notifications or not.

The monitoring mode is usually set before the MonitoredItem is being connected. Changing the monitoring mode of a connected MonitoredItem will immediately change its state in the server.

Parameters:

monitoringMode

The monitoring mode of the MonitoredItem.

Note:

Default value is EnumMonitoringMode_Reporting

To modify the monitoring mode of multiple items, please use the methods Client::Subscription::setMonitoredItemsMonitoringMode() or Client::Subscription::setAllMonitoredItemsMonitoringMode().

EnumStatusCode SoftingOPCToolbox5::Client::MonitoredItem::setSamplingInterval(double samplingInterval)

Sets the sampling interval of the MonitoredItem. The server may revise this value to a more appropriate value which can be retrieved by getRevisedSamplingInterval() after the MonitoredItem has been connected.

The value 0 indicates that the server should use the fastest practical rate. The value -1 indicates that the publishing interval of the subscription shall be used for the sampling interval.

In case of an event monitored item this value must be 0.

Parameters:

samplingInterval

The sampling interval in milliseconds

Note:

The default value is 0.

Please use Client::Subscription::updateChangedMonitoredItems() to apply changes to the sampling interval when the MonitoredItem is already connected (isConnected() == true).

OTUInt32 SoftingOPCToolbox5::Client::MonitoredItem::getQueueSize() const

Returns the maximum number of data change or event notifications that a MonitoredItem may hold in its queue. Notifications are dequeued by the Subscription every publishing interval.

See also:

setQueueSize()

EnumStatusCode SoftingOPCToolbox5::Client::MonitoredItem::setQueueSize(OTUInt32 queueSize)

Sets the maximum number of data change or event notifications that a MonitoredItem may hold in its queue. Notifications are dequeued by the Subscription every publishing interval.

The queue of notification messages allows clients to collect more than one data value change or event for the same node with one publish call.

If the server wants to enqueue a notification message and the queue has reached the maximum number of elements, then depending on the DiscardOldest parameter (see setDiscardOldest()) either the oldest or the latest notification is discarded.

Parameters:

queueSize

The maximum number of data change or event notification messages that the MonitoredItem shall keep in its queue before the queue is emptied by the Subscription during publishing. If the value 0 is provided, the server will use the default queue size.

Note:

The default value is the value 0 which means that the server will use the default queue size.

Please use Client::Subscription::updateChangedMonitoredItems() to apply changes to the queue size when the MonitoredItem is already connected (isConnected() == true).

bool SoftingOPCToolbox5::Client::MonitoredItem::getDiscardOldest() const

Returns whether the oldest or the latest notification message is discarded when the queue of notification messages is full and a new notification is to be enqueued.

Returns true, if the oldest notification message is discarded. Returns false, if the latest notification message (i.e. the new notification to enqueue) is discarded.

See also:

setDiscardOldest()

Parameters:

discardOldest

If true, the oldest notification message in the queue is discarded. (Default) If false, the new notification message to enqueue is discarded.

Sets the data change filter that decides whether a data change is to be enqueued (and reported) or not. The data change filter defines

•which DataValue attributes (value, quality/status, timestamp) must have changed for the data change to be enqueued.
•an optional deadband for analog values.

Parameters:

The data change filter

Note:

Please use Client::Subscription::updateChangedMonitoredItems() to apply changes to the data change filter when the MonitoredItem is already connected (isConnected() == true).

The data change filter must only be specified for data change- (and not event-) monitored items.

virtual void SoftingOPCToolbox5::Client::MonitoredItem::onError(EnumStatusCode errorCode) [virtual]

This method is called whenever an error concerning this MonitoredItem occurs. Applications should override this method to catch errors that occur due to this monitored item.

Typical contexts in which this callback is called are errors in following situations:

•connecting the monitored item on the server node
•deleting a monitored item from the server
•updating the server with the monitored items modifications
•setting the monitoring mode for monitored items
•setting triggered monitored items (for the items to be added or removed)

Parameters:

errorCode

The status code of the error that occurred
