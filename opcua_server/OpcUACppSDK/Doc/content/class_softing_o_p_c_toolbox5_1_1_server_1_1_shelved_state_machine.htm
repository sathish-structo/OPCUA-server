ShelvedStateMachine Class Reference

#include "ServerShelvedStateMachine.h"

 

The ShelvedStateMachine class defines a state machine that represents an advanced alarm filtering model. This StateMachine defines three states: Unshelved, TimedShelved and OneShotShelved.

EnumStatusCode SoftingOPCToolbox5::Server::ShelvedStateMachine::shelve()

Shelves the alarm using one-shot shelving. The one-shot shelving clears automatically when an alarm returns to an inactive state.

EnumStatusCode SoftingOPCToolbox5::Server::ShelvedStateMachine::timeShelve(double duration)

Shelves the alarm using timed shelving. In timed shelving, a user specifies that an alarm is to be shelved for a fixed period of time.

Parameters:

duration

The time (in milliseconds) that indicates how long the alarm is to be shelved

EnumStatusCode SoftingOPCToolbox5::Server::ShelvedStateMachine::unshelve()

Unshelves a shelved alarm.

double SoftingOPCToolbox5::Server::ShelvedStateMachine::getUnshelveTime() const

Returns the remaining time in milliseconds until the alarm is automatically being unshelved.

virtual EnumStatusCode SoftingOPCToolbox5::Server::ShelvedStateMachine::setUnshelveTime(double unshelveTime) [virtual]

Sets the remaining time in milliseconds until the alarm is automatically being unshelved.

•For one-shot shelving, the unshelve time should be initialized with the value of the 'MaxTimeShelved' property
•For timed shelving, the time should be initialized with the provided period of time
