The installation directory of the SDK, hereafter referred to as <InstallDir>, contains the following directories:

Directory location

Description

<InstallDir>/Doc

Location of the SDK's documentation

Location of SDK libraries, test applications and programming samples.

Workspace containing projects for delivered sources (StructoAutomoldServer and TestClient)

Workspace containing projects for delivered sources (StructoAutomoldServer and TestClient) using DKM (Downloadable Kernel Module) configurations

<InstallDir>/Source/Workspace7

Workspace containing various batches and shell scripts to use SDK projects

<InstallDir>/Tools

Location of additional utilities (trace viewer)

SDK Components (Binary)

For VxWorks, the SDK's components are shipped as static libraries (archives):

•libtb5cpp.a: The TB5CPP component (C++ API)
•libtb5ot.a: The TB5OT component (SDK core)
•libtb5stack.a: The TB5Stack component (OPC UA communication stack)
•libtb5util.a: The TB5Util component (Utilities library)
•libcrypto.a: The OpenSSL component
•libxml2.a: The libxml2 XML parser component

Libraries that can be used by real-time process (RTP) applications are available for the following CPU architectures and compilers (hereafter referred to as <Arch>):

•ARMv7 / Wind River Diab (ARMARCH7diab_RTP)
•Intel Nehalem / GNU GCC (NEHALEMgnu_RTP)
•PPC32 / Wind River Diab (PPC32diab_RTP)
•PPC32 (soft-float) / Wind River Diab (PPC32sfdiab_RTP)
•Wind River Target Simulator (x86) / GNU GCC (SIMPENTIUMgnu_RTP)

The delivered binaries are built using VxWorks 6.9.4.12 and VxWorks 7 SR0650.
Usually the binaries are compatible with newer VxWorks version, in may times also with older version.
The binaries are built for RTP; if you need DKM binaries (or different build architectures, CPUs or compilers), please use the source code version to build it by yourself.

VxWorks 7 builds are always based on a VSB; we deliver binaries for VSB build on CPU "SIMNT" only.
All other VSBs need source code version.

The libraries can be found at the following locations:

Directory location

Description

<InstallDir>/Source/lib/VxWorks<version>/<Arch>/NonDebug

Location of the static libraries compiled for the architecture <Arch>.

<InstallDir>/Source/lib/VxWorks<version>/<Arch>/Debug

Location of the static libraries compiled for the architecture <Arch>. Applications linked with this set of libraries support additional trace options for extended diagnosis information but are larger and slower than libraries built without this option.

SDK Components

The TB5CPP component of the SDK that provides the C++ API is delivered as binary. The needed header files can be found at the following locations:

Directory location

Description

<InstallDir>/Source/api/include

Location of the class header files for the C++ API..

<InstallDir>/Source/api/include/ModelCompiler

Location of the header files that define the public API of the Model Designer components.

<InstallDir>/Source/Core/include

Location of the class header files for the TB5OT component.

Test Applications

The SDK comes with source code for applications that are used to verify the installation of the SDK:

•CPPTestClient: C++ test client application.
•CPPTestServer: C++ test server application.

The following table depicts locations associated with these applications.

Directory location

Description

<InstallDir>/Source/Apps/Test/Cpp/StructoAutomoldServer

Location of the class header and implementation files for the test server application

<InstallDir>/Source/Apps/Test/Cpp/StructoAutomoldServer/TestFiles

Location of test files and test folders to test file and file directory access functionality of the server

<InstallDir>/Source/Apps/Test/Cpp/TestClient

Location of the class header and implementation files for the test client application.

<InstallDir>/Source/Apps/Test/Cpp/TestClient/TestFiles

Location of test scripts that drives the test client application.

Certificate Management

The following table describes the directories that contain scripts for managing application instance certificates.

Directory location

Description

<InstallDir>/Source/PKI

Location of batch files / scripts that are used to create a demo certificate store.

<InstallDir>/Source/PKI/batches

Location of batch files / scripts that are required by the batch files described above.

<InstallDir>/Source/PKI/all_certificates

<InstallDir>/Source/PKI/sample_client

<InstallDir>/Source/PKI/sample_server

<InstallDir>/Source/PKI/test_client

<InstallDir>/Source/PKI/test_server

Location of certificate stores. Please note that these locations are only available after creating the certificate stores using the batch files / scripts described above.

For further details see Setting up a Certificate Store

Note: A simulator target has to be build; the pre-installed simulator target cannot be used since "POSIX thread scheduled in RTPs" has to be enabled within the image project.

Note: For VxWorks 7 only libraries for Windows simulator are available.
