Installing the SDK for use with Linux

Please execute the provided shell script InstallOpcUaCppSDKLinux6.00.0.sh.

Note: The installer script must not be copied as text file. Actually at the end of the script a binary file is appended.

 

Specify the installation directory and optionally provide the source code licence key.
If the installation directory exists, it will be deleted.

Optional options of script:

Option

 

-d installdir

Directory where the SDK will be installed. If not provided, directory has to be specified interactive.

-f

Force overwriting existing installation directory without asking.

-p productkey

Product key to install source code (use "" for binaries only).

-m [i386 | x86_64]

Machine type of system running the script (Intel 32 bit or 64 bit). Used for installer executable only, installed files are identical.

If not specified, the script tries to identify the system by itself.

This option should be used only, if installer script cannot detect machine type by its own.

The SDK is built to provide a maximal compatibility to common Linux distributions and is tested to run on Ubuntu 16.04, Ubuntu 18.04 and Ubuntu 20.04 on Intel CPU.
Binaries for ARM are also delivered for various gcc version..

Note: Most distributions running on Intel CPU should work with the delivered binaries of the SDK, but are not tested.

Note: If the delivered binaries do not work on your Linux distribution or you are developing an application for a CPU where binaries are not delivered, the SDK libraries may need to be recompiled.
You can do this by your own, if you have bought SDK source code license.

 

Installation Directory

The installation directory of the SDK, hereafter referred to as <InstallDir>, should not contain special characters like blanks, "#", ...
The SDK itself works well in such installation paths; in case of source code license you may run into issues if OpenSSL does not support these special characters.

The <InstallDir> contains the following directories:

Directory location

Description

<InstallDir>/Doc

Location of the SDK's documentation

<InstallDir>/Source

Location of SDK libraries, test applications and programming samples.

<InstallDir>/Tools

Location of additional utilities (trace viewer)

SDK Components (Binary)

For the Linux platform, the SDK's components are shipped as static libraries (archives) and shared objects (so). The libraries are available for both 32 bit and 64 bit Intel CPU architectures and ARM.

•libtb5cpp.a: The TB5CPP component (C++ API)
•libtb5ot.a: The TB5OT component (SDK core)
•libtb5stack.a: The TB5Stack component (OPC UA communication stack)
•libtb5util.a: The TB5Util component (Utilities library)
•libcrypto.a: The OpenSSL component for cryptography
•libssl.a: The OpenSSL component for SSL/TLS
•libxml2.a: The libxml2 XML parser component
•libToolkitUA.so.6.00.0: All SDK components above (except libcrypto, libssl and libxml2) as a shared library
•libToolkitUA.so: Link to the specific shared library

The libraries can be found at the following locations. The placeholder <PlatformDir> is an identifier for the target platform, e.g. i386 and x86_64 for for 32 bit and 64 bit Intel binaries and arm for binaries for ARM. The placeholder <Gcc> resolves to the used GCC version including a possible cross compiler prefix.

Currently delivered gcc versions are gcc5, gcc7 and gcc9.

Directory location

Description

<InstallDir>/Source/lib/linux/<PlatformDir>/<Gcc>/release

Location of the static libraries.

<InstallDir>/Source/lib/linux/<PlatformDir>/<Gcc>/release-shared

Location of the shared library.
