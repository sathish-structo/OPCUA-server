Data Type Overview

Top  Previous  Next

OPC UA defines four kind of data types:

•Built-in data types
•Simple data types
•Enumeration data types
•Structured data types

Built-in data types are a fixed set of fundamental data types that are defined by the OPC UA specification. The built-in types are depicted in the table below.

Simple data types are sub-types of built-in data types. They imply additional semantics but have the same representation as built-in types on the wire. The simple data type "UtcTime", for example, derives from the built-in data type "DateTime" and is used to define timestamps in UTC time. Therefore, OPC UA applications have to use the DateTime class to define UTC timestamps. Another example of a simple data type is the "Duration" data type. This data type derives from the built-in "Double" type and is used to represent a time span in milliseconds.

Enumeration data types are encoded as zero-based Int32 values on the wire. The OPC UA C++ Server & Client SDK provides C++ enumerations for all well-known enumeration types (see the "Enumerations" topic for further reference).

Structured data types are used to represent structured information that cannot be represented as built-in type. The SDK provides C++ classes for well-known structured data types. The "ReadValueId" data type, for example, holds details of a read request and is represented by the C++ class ReadValueId.

Union data types are custom data types that allow the selection of only one of the possible fields. Unions are encoded as a UInt32 switch field followed by the selected field. The switch field is the index of the available union fields starting with 1.
