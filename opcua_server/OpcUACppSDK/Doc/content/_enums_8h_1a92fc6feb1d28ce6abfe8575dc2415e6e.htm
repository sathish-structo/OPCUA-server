EnumCertificateCheck Reference

Top  Previous  Next

Defines bit masks what errors shall be ignored during validating the certificate.

EnumCertificateCheck_All = 0x00000

No validation errors are ignored, i.e. every validation error causes the certificate to be rejected.

EnumCertificateCheck_SuppressCertificateValidityPeriodCheck = 0x00040

The certificate is not rejected if the current time is not within the period specified in the certificate.

EnumCertificateCheck_SuppressCrlValidityPeriodCheck = 0x00080

The certificate is not rejected if the current time is not within the period specified in the revocation list.

EnumCertificateCheck_SuppressCrlNotFoundError = 0x00100

The certificate is not rejected if a CRL is not found.

EnumCertificateCheck_AllowPartialChain = 0x40000

The certificate is not rejected if the issuer list and trust list don't contain the entire certificate chain to the root certificate.

The issuer list only needs to cover the certificate chain from the leaf up to the next trusted certificate. No chain validation takes place if the leaf certificate is in the trust list. Note: This option is not compliant to the OPC UA specification and should only be used to resolve compatibility problems
