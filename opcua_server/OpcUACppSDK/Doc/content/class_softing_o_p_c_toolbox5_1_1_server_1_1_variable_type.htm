#include "ServerVariableType.h"

 

The VariableType class is used to define a VariableType node in the address space of a server.

A VariableType provides definitions for a Variable in the address space, i.e. it defines a blueprint for variables (instances of the VariableType). Clients use the VariableType to learn about the properties of a variable.

All variable types finally derive from the variable type "BaseVariableType". The well-known "BaseDataVariableType" is the base type for all variables that represent values of objects. The well-known "PropertyType" is the base type for all variables that describe the characteristics of a node ("properties").

Note:

Each Variable must be assigned to a VariableType using BaseNode::init().

virtual EnumStatusCode SoftingOPCToolbox5::Server::VariableType::validateValue(const IValue & value, const INumericRange * pIndexRange) [virtual]

Validates whether an input value is valid to be used for the Value attribute of this VariableType.

The method validates that

•the data type matches the configured data type
•value rank and array dimensions match configuration
•value meets restriction on range, enumeration, ...

This method may be called when handling a write request to the Value attribute. Derived classes may add additional validation steps.

Parameters:

dataValue

The value to validate

pIndexRange

The optional index range when only a subset shall be updated. Use NULL or an empty NumericRange for no index range checks.

OTInt32 SoftingOPCToolbox5::Server::VariableType::getValueRank() const

Returns whether the variable's value is an array and if, how many dimensions the array has.

Possible return values:

•>= 1: Indicates that the variable contains an array value with this number of dimensions
•ValueRank_OneOrMoreDimensions: Indicates that the variable contains an array value of one or more dimensions
•ValueRank_Scalar: Indicates that the variable contains a scalar value
•ValueRank_Any: Indicates that the variable contains any value, scalar or array of any dimension
•ValueRank_ScalarOrOneDimension: Indicates that the variable contains a value of scalar or one dimensional array

See also:

setValueRank()

EnumStatusCode SoftingOPCToolbox5::Server::VariableType::setValueRank(OTInt32 valueRank)

Sets whether the variable's value is an array and if, how many dimensions the array has.

Possible values:

•>= 1: Indicates that the variable contains an array value with this number of dimensions
•ValueRank_OneOrMoreDimensions: Indicates that the variable contains an array value of one or more dimensions
•ValueRank_Scalar: Indicates that the variable contains a scalar value
•ValueRank_Any: Indicates that the variable contains any value, scalar or array of any dimension
•ValueRank_ScalarOrOneDimension: Indicates that the variable contains a value of scalar or one dimensional array

Note:

If the array dimensions don't match to the new value rank, then the array dimension are automatically updated to matching dimensions without length limitation.

See also:

setArrayDimensions

std::vector< OTUInt32 > SoftingOPCToolbox5::Server::VariableType::getArrayDimensions() const

Returns a field of integer values describing the length of each array dimension if the variable has an array value. The number of elements is equal to the result of getValueRank() if getValueRank() returns a positive number. A vector entry with value 0 indicates that the dimension has variable length.

Note:

This method returns an empty vector for scalar values or variable sized dimensions.

EnumStatusCode SoftingOPCToolbox5::Server::VariableType::setArrayDimensions(const std::vector< OTUInt32 > & arrayDimensions)

Sets a field of integer values describing the length of each array dimension if the variable has an array value. The number of elements must be equal to the result of getValueRank() if getValueRank() returns a positive number. A vector entry with value 0 indicates that the dimension has variable length.

Note:

The array dimensions attribute must not be set for scalar variable values or variable sized dimensions. If the value rank does not match to the new array dimensions, then the value rank is automatically updated to the exact dimension.
