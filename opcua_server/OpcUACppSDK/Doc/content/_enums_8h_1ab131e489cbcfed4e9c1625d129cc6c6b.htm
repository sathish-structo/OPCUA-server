EnumHostnameResolution Reference

Top  Previous  Next

Configuration how a symbolic hostname in an URL shall be resolved.

Whenever an URL with a symbolic hostname is given (either in Client::Session, Application::GetEndpointsFromServer, ...) the SDK shall modify the URL to replace the symbolic name by a real IP address or leave it as it is.

If a specific IP address is provided in the URL (e.g. "127.0.0.1" or "[::0]") the setting is ignored and the given IP address is used directly.

EnumHostnameResolution_PreferIpV4 = 1

The symbolic hostname shall be resolved to an IPv4 address.

If no IPv4 address is found, then an IPv6 address is used.

EnumHostnameResolution_PreferIpV6 = 2

The symbolic hostname shall be resolved to an IPv6 address.

If no IPv6 address is found, then an IPv4 address is used.

EnumHostnameResolution_DoNotConvert = 3

The symbolic hostname shall NOT be resolved in the URL.

Note:

This option leaves the hostname as it is provided by client or server. This may be necessary, if a server compares the provided URL when it is connected. A resolution however is needed whenever a communication socket is needed. If resolution to both IPv4 and IPv6 is possible, it is unpredictable which resolution will be done. In case of such servers, it might be necessary do disable either IPv4 or IPv6 to get a predictable resolution.

EnumHostnameResolution_Invalid = -1

Invalid value for hostname resolution.
