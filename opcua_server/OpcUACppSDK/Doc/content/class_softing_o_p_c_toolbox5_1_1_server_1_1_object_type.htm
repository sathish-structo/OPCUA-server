ObjectType Class Reference

#include "ServerObjectType.h"

 

The ObjectType class is used to define an ObjectType node in the address space of a server.

An ObjectType provides definitions for an Object in the address space, i.e. it defines a blueprint for objects (instances of the ObjectType). Clients use the ObjectType to learn about the references, components, and properties that an Object instance provides.

All object types finally derive from the object type "BaseObjectType".

Example: The ObjectType "FolderType" is a well-known object type that a server may use to organize information (e.g. variables and/or objects).

Note:

Each Object must be assigned to an ObjectType using BaseNode::init().

EnumStatusCode SoftingOPCToolbox5::Server::ObjectType::setIsAbstract(bool isAbstract)

Indicates whether objects of this type may be exposed in the address space of the server. If the object type is abstract, an object of this type must not be exposed in the address space. However, the object type may still be used as event type in event notifications (see EventNotification).

virtual void SoftingOPCToolbox5::Server::ObjectType::handleCallRequest(Server::MethodCallTransaction * transaction, Server::MethodCallRequest * request) [virtual]

Callback method that is triggered when a UA method is invoked in context of this object type by a client. This method should be overridden by server applications.

Parameters:

transaction

The transaction which contains the single call request

request

The call request which shall be handled

Note:

The request is pre-processed by the SDK. In case of errors (e.g. access restrictions, input arguments, ...) the status code of the request is set. In case of invalid input arguments the input argument results is set by SDK.

When overriding this method you mustn't use blocking calls within this method since this blocks the UA communication.

This callback won't be invoked for methods where the callback Method::handleExecute() is overridden, unless the overriding implementation invokes Method::executeRequest().

The method does not have a return value - us request->setStatusCode to pass a result to the client.
