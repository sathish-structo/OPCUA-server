Programming Tutorial

The tutorial presented in this section introduces you to the most essential features that you will need to develop your OPC UA-enabled application.

The tutorial describes the creation of two applications, a client and a server application. Each step highlights a different feature of OPC UA and the SDK. Sample applications that demonstrate those features accompany each step. All sample applications are based upon each other. The tutorial starts out with skeleton code that contains just the minimum to run an OPC UA application and ends up with a full-blown client or server application.

In most cases you will be interested in only one side of the communication, either client or server. To get the most out of the sample applications, we recommend to run the full-blown server sample when testing your client, and vice-versa, run the full-blown client sample when testing your server.

Where to Find the Sample Application Projects and How to Start them

Windows:

•For Visual Studio 20[10|13|15|17|19] there is one solution file that contains all sample application described in this tutorial (see Windows Installation Directory Structure for where to find the tutorial projects (Programming Samples)). The files are named Tutorial_20[13|15|17|19|22].sln.
oOpen the appropriate file with Visual Studio.
oCompile and start/debug the desired projects with Visual Studio.

Linux:

•Please follow the following instructions to compile and run the sample applications on Linux:
oSee Linux Installation Directory Structure for where to find the tutorial projects (Programming Samples) and executable files (the same path as for the Test Applications).
oOpen a command shell.
oChange the working directory to the path of the tutorial.
oCompile all tutorial samples by calling "make -f linux_gcc.mak" or "make -f linux_gcc.mak BUILD_TARGET=debug" to include debug symbols. Each sample provides its own linux_gcc.mak makefile and can also be compiled individually.

VxWorks:

•For VxWorks 6.9 simply import the existing project files below .../Source/Apps/Samples into workspace via workbench (both tutorial and samples).
•For VxWorks 7 all delivered projects are based on a VSB (VxWorks Source Build) project, which is based on a BSP (Broad Support Package) and / or a CPU.
The delivered libraries are based on a VSB created for Windows simulator.
To create the tutorial projects, invoke the "AddTutorialProjects" batch / shell script. (For using the programming samples invoke "AddSampleProjects")
oNote: Projects "Refrigerator" and "Utilities" have to be compiled first
oNote: The libraries produced with these projects are created below the workspace
However the tutorials expect them to be locates at <install_dir>/Source/lib/<VSB_NAME>_<DKM or RTP>/...
Please copy them to the according locations after compilation.
 
Alternative: Edit the ".wrmakefile" of the projects by adding a deployment for your VSB name. The section can read like
deploy_output ::
       mkdir -p ../../../lib/VxWorks7/$(BUILD_SPEC)/$(MODE_DIR)
       cp $(PROJECT_TARGETS) ../../../lib/VxWorks7/SIMNT_RTP/$(MODE_DIR)/
 

How to use the Tutorial

It is highly recommended to wade through the SDK introduction to clarify the basic concepts before using this tutorial section. It is also recommended to have a look at the OPC UA Introduction if you are new to OPC UA.

After reading the notes for each step, please have a look at the sample code or use a debugger to step through the code. The source code contains additional explanations on how to use the different UA services. Some code snippets in the feature description may appear a bit more complicated in the code due to error handling.
