Publisher Configuration

Top  Previous  Next

A PubSub Publisher consists of four main classes which need to be configured:

•PublishedDataSet
•DataSetWriter
•WriterGroup
•Connection

Each of those main classes might involve additional data type classes for its configuration.

PublishedDataSet

The PublishedDataSet describes the data to publish, e.g. the used data types and the order of the fields.

Note: The PublishedDataSet is a concrete base type, you can use also a specialized subtype instead.

See Configure PublishedDataSet

 

DataSetWriter

Every DataSetWriter is bound to one PublishedDataSet and describes how the PublishedDataSet is written into a DataSetMessage.

For example, it describes if the value is encoded as raw data, as Value or as DataValue and if all values shall be notified or only the values where the content has changed.

Note: One PublishedDataSet can be assigned to several DataSetWriters, e.g. to encode the same content in a different manner or to send the same content via a different WriterGroup and Connection to a different network.

See Configure DataSetWriter

 

WriterGroup

The WriterGroup is a subtype of the abstract Group, specialized for Publishers.

It groups a list of DataSetWriters and describes how to put the DataSetMessage of the DataSetWriters into one NetworkMessage (e.g. the UDP packet on the wire) and when to send the notifications.

The WriterGroup configures parameters like the publishing interval and the used security settings to sign or encode the NetworkMessage (not yet supported).

Note: One DataSetWriter can be assigned to only one WriterGroup.

See Configure WriterGroup

 

Connection

The Connection describes how to send NetworkMessages to the subscribers and groups a list of WriterGroups, whose NetworkMessages shall reach the same target(s).

It configures parameters like the used protocol and the network address.

Note: One WriterGroup can be assigned to only one Connection.

Note: The Connection class is also used for Subscribers.

See Configure Connection
