Deployment of Applications Built with the SDK

The following SDK libraries need to be shipped along with your application:

For a 32 bit application:

•TB5CPP.dll: The TB5CPP component (C++ API)
•TB5OT.dll: The TB5OT component (SDK core)
•TB5STACK.dll: The TB5Stack component (OPC UA communication stack)
•TB5UTIL.dll: The TB5Util component (Utilities library)
•libcrypto-1_1.dll: The OpenSSL component
•libssl-1_1.dll: The OpenSSL component
•libxml2.dll: The libxml2 component

For a 64 bit application:

•TB5CPPx64.dll: The TB5CPP component (C++ API)
•TB5OTx64.dll: The TB5OT component (SDK core)
•TB5STACKx64.dll: The TB5Stack component (OPC UA communication stack)
•TB5UTILx64.dll: The TB5Util component (Utilities library)
•libcrypto-1_1.dll: The OpenSSL component
•libssl-1_1.dll: The OpenSSL component
•libxml2x64.dll: The libxml2 component

Please verify that the SDK DLLs can be loaded by your application (see Dynamic-Link Library Search Order for further reference).

The pre-compiled SDK libraries expect the C Run-Time (CRT) Library to be present at runtime. Therefore, you need to deploy the appropriate Microsoft Visual C++ Redistributable packages. The SDK libraries are compiled with the following compiler versions:

•Microsoft Visual Studio 2022 Version 17.2.1
•Microsoft Visual Studio 2019 Version 16.11.15
•Microsoft Visual Studio 2017 Version 15.9.48
•Microsoft Visual Studio 2015 Version 14.0.25420.01 Update 3
•Microsoft Visual Studio 2013 Version 12.0.21005.1 REL

Visual Studio versions 2015 and newer are using Universal CRT which is always available as a Windows component; separate installation of Redistributable packages is not required.
