#include "ServerConfiguration.h"

 

The ServerConfiguration class is used to represents the configuration of a server which supports GDS push management.

There is always exactly one instance in the server address space. Due to this fact an application cannot create other instances; the only creation is done by SDK on initializing or via XML import. If you need to overwrite any method handling, you need to instantiate your derived class via Creator::createServerConfiguration.

The ServerConfiguration contains mandatory properties to describe how GDS can access the server:

•ServerCapabilities
•SupportedPrivateKeyFormats (only PEM formats are supported by SDK)
•MaxTrustListSize
•MulticastDnsEnabled (multicast DNS is not supported by SDK)

The ServerConfiguration handles UA methods which are invoked by GDS:

•UpdateCertificate
•ApplyChanges
•CreateSigningRequest
•GetRejectedList

Note:

The properties SupportedPrivateKeyFormats and MulticastDnsEnabled are handled by SDK only.

EnumStatusCode SoftingOPCToolbox5::Server::Configuration::setMaxTrustListSize(OTUInt32 maxTrustListSize)

Sets the MaxTrustListSize property of the server configuration.

All trusted and issuer certificates are exposed as a file in a trust list object. This property specifies the maximum of this representation.

Parameters:

maxTrustListSize

Maximum of the file representation. Default is 64 kB (65535 byte).

virtual void SoftingOPCToolbox5::Server::Configuration::handleCallRequest(Server::MethodCallTransaction * transaction, Server::MethodCallRequest * request) [virtual]

Callback method that is triggered when a UA method is invoked in context of this object by a client. This method should be overridden by server applications.

Parameters:

transaction

The transaction which contains the single call request

request

The call request which shall be handled.

Note:

The request is pre-processed by the SDK. In case of errors (e.g. access restrictions, input arguments, ...) the status code of the request is set. In case of invalid input arguments the input argument results is set by SDK.

When overriding this method you mustn't use blocking calls within this method since this blocks the UA communication.

This callback won't be invoked for methods where the callback Method::handleExecute() is overridden, unless the overriding implementation invokes Method::executeRequest().

The method does not have a return value - us request->setStatusCode to pass a result to the client.

Reimplemented from SoftingOPCToolbox5::Server::Object.
