OPC UA C++ Server & Client SDK V6.00

Certificate Management

Top  Previous  Next

See the OPC UA - Certificate Management for a common description what the certificate management of OPC UA supports.

The OPC UA C++ Server & Client SDK V6.00 offers support for UA applications to interact with a Certificate Manager like the GDS.
This includes automated management support for certificates and trust lists of application instance certificates as well as some manual support functions so that an application is able to maintain any other certificates or trust lists by itself.

Note: Even though a GDS may be able to provide individual HTTPS certificates, the SDK does not support to use them. The OPC UA C++ Server & Client SDK V6.00 reuses the the application instance certificates for HTTPS.

 

The SDK supports both certificate management models and requires some different configuration settings to enable the Pull or the Push management models or both.

•Pull Management: Describes the SDK functionality and required settings related to the Pull management model where the UA application takes the active role to retrieve the certificates and trust lists from the Certificate Manager.
•Push Management: Describes the SDK functionality and required settings related to the Push management model where a Certificate Manager takes the active role to distribute certificates and trust lists.
